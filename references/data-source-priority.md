# 数据获取工具选择优先级 (Data Source Priority)

统一约定所有复盘/推送/分析技能**取数**时的工具选择顺序。各技能 SKILL.md 凡涉及"行情、财务、资金流向、消息、公告"的获取均引用本文件。

## 核心机制:读记忆里的具体配置,不再每次临场推断

旧做法只按"工具能力类别"做抽象描述、不绑定具体工具名,导致每次取数都要临场推断"哪个算专业工具、该走谁",效果不稳。**现在改为读一份固化的具体配置**:

- 由 `ashare-data-source-config` 技能**探测本地实际装了哪些取数工具**,由 AI 一次性编排出"数据类型 → 有序工具链(含兜底)"的具体配置,写入 auto-memory 的 `data-source-config.json`(结构见该技能 `references/config-schema.md`)。
- 各复盘/分析技能取数时**直接读这份配置照做**:按要取的数据类型查对应桶的 `primary` → 取不到依次走 `fallbacks`,并遵守该桶 `notes` 里的口径/兜底声明要求。

### 取数前的配置准备(每次复盘启动时)

1. **读配置**:读 auto-memory 的 `data-source-config.json`。
2. **配置不存在** → 接力调用 `ashare-data-source-config`(完整探测模式)先建立配置,再继续。
3. **配置存在** → 按 `ashare-data-source-config` 的"轻量校验模式"快速核对:配置里标为 `primary` 的关键专业工具此刻是否仍在当前 skill/MCP 列表、运行时依赖是否仍在、`next_review_due` 是否已过期。任一不满足 → 接力调用该技能重新编排;都满足 → 直接用。
4. **按配置取数**:对每类要取的数据,查 `routing` 对应桶,依 `primary`→`fallbacks` 顺序取,取到即停。

> 轻量校验要快——常规只读配置+比对工具列表,不必每次对每个工具做试探调用,以免拖慢复盘启动。

## 总原则(配置编排与兜底时都遵守)

**先用专业的,再用通用的。** 同一份数据若专业金融工具与通用联网搜索都能拿到,**永远优先专业工具**;联网搜索只在专业工具拿不到、不覆盖该市场、或补充时效/广度时上场。这条原则已固化进配置的 `routing` 顺序;配置缺失需临场取数时,仍按此原则择优,事后补跑配置技能。

## A. 结构化数值类(行情、K线、财务、资金流向、估值、股东、指数成分、宏观指标等)

要的是**精确数字**,对来源权威性和口径一致性敏感。按配置对应桶(`cn_equity_quote`/`cn_equity_fundamental`/`hk_us_equity`/`fund_index_bond`/`cn_tech_daily`/`macro_indicator`)的工具链取数,顺序为:

1. **用户本次明确指定的数据源**(临时覆盖配置)。
2. 配置里该桶的 `primary`——通常是专业金融数据工具/MCP。
3. 该桶的 `fallbacks`——本地取数脚本/数据包,或其它专业源。
4. **通用联网搜索——仅兜底**:前面都拿不到时才用,**只采信定性信息(涨跌/方向/大致区间),精确数字不当可靠数据填报**,并注明"来自公开搜索、未经专业源核对"。

> 结构化数值能用专业工具就绝不用搜索凑数字;专业工具字段缺失时按各技能降级规则如实声明,不编造。

## B. 消息资讯类(新闻、政策、公告、研报、事件、舆情)

要的是**覆盖面 + 时效 + 可核对来源**。按配置 `news_policy` / `announcement_doc` 桶取数,**默认多工具组合、交叉印证,不允许只用单一通用搜索**:

1. **专业金融工具的资讯/公告/事件接口**(配置 `primary`)——拿带明确来源、可核对的官方公告、监管信息、个股异动事件,作消息面"权威底座"。
2. **通用联网搜索补时效与广度**(配置 `fallbacks`)——覆盖专业工具未收录的最新进展、市场解读、全球事件。
3. **交叉印证**:以专业/官方来源事实为准,搜索补背景;冲突时以更权威、更原始来源为准并保留来源标注。
4. 环境里无任何专业资讯工具时才退化为"仅联网搜索",此时须注明"未经专业资讯源交叉核对",提示可靠性局限。

## C. 专业工具不覆盖的市场(如日股、欧股、汇率、隔夜外盘等)

部分专业工具有明确不覆盖的市场(例如 Wind 不含日股/欧股/汇率/期货)。这类数据走配置 `overseas_uncovered` 桶:**无专业源,只能联网搜索,且只采信定性信息(涨跌/方向/大致区间),精确数字注明未经专业源核对**。注意:港股恒指、美股美指等若专业工具覆盖,仍应走对应专业桶,不要误归入此桶。

## D. 数据来源标注(Source Attribution)

所有复盘/推送/分析技能生成的报告,**必须在末尾包含"数据来源"小节**,标注报告中各数据块的实际来源工具,实现数据可溯源。

### 标注格式

统一使用表格:

```
### 数据来源
| 数据块 | 来源工具 | 备注 |
|--------|---------|------|
| [数据块描述] | [工具名] | [桶名, primary/fallback/交叉印证等] |
```

- **数据块粒度**:按报告的自然分区标注(如"隔夜美股行情"、"A股全天行情"、"技术指标计算"等),不逐条数字标注。
- **工具名**:使用 `data-source-config.json` 中 `tools[].id` 的值(如 `wind_mcp`、`web_search`);若同时使用专业工具+搜索交叉印证,写两者并用。
- **如实记录降级**:实际取数时若 primary 不可用走了 fallback,须在备注中写明(如"Wind MCP 不可用,降级为 akshare 本地脚本")。

### 已纳入来源标注的技能

各技能 SKILL.md 输出格式模板中已包含"### 数据来源"小节,执行时取数完成后按实际使用的工具填充。各技能对应的典型数据块见各自模板。

## 与各技能降级规则的关系

- 本文件管"优先用哪个工具取";各技能 SKILL.md 既有的"数据缺失时如实声明、绝不编造"降级规则继续有效,两者叠加:先按配置择优取数,取不到时按降级规则诚实处理。
- 数据契约(需要哪些字段、什么口径)仍由各技能自定,本文件与配置都不改变"要拿到什么"。
