# 技术面深度共享指南 (Technical Depth Guide)

四个复盘技能（`ashare-morning-brief` / `ashare-intraday-review` / `ashare-evening-review` / `ashare-weekly-review`）统一引用本指南，把"技术面深度"作为复盘内容的一部分，避免报告停留在消息面流水账。本指南规定：何时调用技术引擎、三类资产怎么调、引擎输出怎么读、各时段技术深度配方、大盘与板块的深度写法、条件式情景的写法模板、以及数据不可用时的降级规则。

技术深度的边界统一遵守 `boundaries.md` §7。本指南只解决"怎么做深"，不放宽边界。

## 一、核心原则：把"数字"翻译成"状态+情景"，不是甩数字

报告浮于表面的根因是：要么只有消息面没有技术面，要么有技术面却只是罗列"RSI=72、MACD金叉"这种数字。深度来自**翻译**：

- **看图说话**：把均线排列、量价配合、指标位置用自己的话讲成"当前处在什么技术状态"（例：站上所有均线且多头排列、量能温和放大→趋势健康；但 RSI 进入超买、乖离率偏大→短期有透支风险）。
- **对称情景**：用"若…则…"把后续的两个方向都说清楚（例：若放量站稳 MA20 则上行趋势延续得到确认；若跌破 MA20 且 MACD 死叉则需警惕回调），而不是单边猜涨跌。
- **纪律校准**：由于持仓记录已不再维护止盈/止损/加仓线，不再执行纪律线距离计算。技术位（支撑/压力）仅作为"当前处于什么技术状态"的客观参考，不用于判断是否应操作。
- **alpha vs beta**：用相对强弱（个股 vs 大盘/板块）区分"个股自己的事"还是"跟大盘一起动"。

## 二、技术引擎调用方法（复用 `ashare-technical-analysis` 引擎）

引擎在 `.claude/skills/ashare-technical-analysis/scripts/` 下，两步走：先取数据，再 `compute_indicators.py` 算指标。**从项目根目录运行**，输出统一落到一个复盘缓存目录（建议 `output/_tech_cache/`，用完即弃，不污染最终报告目录）。

**取数方式**（与 `ashare-technical-analysis` 技能步骤1一致）：取数前读 auto-memory 的 `data-source-config.json`（由 `ashare-data-source-config` 技能维护），按 `cn_tech_daily` 桶的路由优先级择源——专业金融 MCP（Wind）→ 本地已安装的 Python 行情包 → 联网搜索兜底（仅采信定性信息）。无论用哪个工具，产出必须满足数据契约：一份标准化日线 CSV，必需列 `date`/`open`/`high`/`low`/`close`/`volume`，前复权，按 date 升序，回溯长度尽量 ≥250 个交易日。任何一环失败按第七节降级处理，不要编造指标数字。

取得 CSV 后，调用 `compute_indicators.py` 计算指标：

**个股（持仓/自选股，默认类型）**：

```bash
python .claude/skills/ashare-technical-analysis/scripts/compute_indicators.py \
  --csv output/_tech_cache/600584_daily.csv \
  --benchmark-csv output/_tech_cache/benchmark_000300.csv \
  --out-json output/_tech_cache/600584_ind.json
```

`--benchmark-csv` 传入沪深300基准 CSV（只需 `date`、`close` 两列），用于相对强弱维度；没有则跳过该维度。

**指数（大盘四大指数）**：指数代码见下表。指数不取基准、不算相对强弱。

```bash

  --csv output/_tech_cache/000001_daily.csv --out-json output/_tech_cache/000001_ind.json
```

## 三、引擎输出 JSON 字段速查

`compute_indicators.py` 输出的关键字段（完整口径见 `ashare-technical-analysis/references/indicators_guide.md`）：

- `latest`：最新交易日各指标值——`close`/`pct_chg`、`ma5/10/20/60/120/250`、`macd_dif/dea/hist`、`rsi6/12/24`、`kdj_k/d/j`、`boll_upper/mid/lower/pctb/bandwidth`、`atr14`、`adx14`/`plus_di14`/`minus_di14`、`vol_ratio`(量比)、`bias20`(乖离率)、`obv_trend_20d`。
- `signals`：离散信号——`ma5_ma20`/`ma20_ma60`/`macd_dif_dea`/`kdj_k_d`(golden_cross/death_cross/none)、`ma_arrangement`(bullish_aligned多头/bearish_aligned空头/mixed)、`price_vs_ma250`(above/below)、`macd_divergence`/`rsi_divergence`(顶背离bearish/底背离bullish)。
- `key_levels`：`support`/`resistance` 各含 `level`(价位)、`touches`(被验证次数)、可能的 `note`。
- `relative_strength`：`alpha_20d`/`alpha_60d`(相对沪深300的超额收益，正=跑赢)；指数或基准缺失时为 null。
- `recent_series`：最近20日精简序列(close/pct_chg/volume/ma20/ma60/rsi12/macd_dif/dea)，用于描述近期走势节奏。

读 ADX 判断"信号可信度"：`adx14`<20 多为震荡市，此时金叉死叉、均线穿越的可靠性低，要在文字里点明"当前趋势强度弱、信号需打折"。

## 四、各时段技术深度配方

四个复盘时段的定位不同，技术深度的"用力点"也不同。核心分工：**晚间复盘是唯一全量跑引擎的时段**，它产出的均线/支撑压力/纪律距离构成一组"技术参考位"；早间推送和盘中复盘**不重复跑引擎**，而是承接昨晚那组参考位，只做"现价 vs 昨晚位"的轻量互动；周复盘把视角拉到中期（周度+长均线）单独跑一次。这样既保证深度，又不让每次复盘都耗时拉数据。

| 时段 | 是否跑引擎 | 技术深度的核心动作 | 承接关系 |
|---|---|---|---|
| 早间推送 morning-brief | 否 | 隔夜外盘定性 + 调出昨晚参考位作为"今日观察基准" | 承接昨晚 evening |
| 盘中复盘 intraday-review | 否 | "技术位互动温度计"：现价 vs 昨晚支撑/压力位 | 承接昨晚 evening |
| 晚间复盘 evening-review | **是（全量）** | 4 大指数 + 每只持仓/自选 全量跑，产出四维深度卡片 | 产出参考位供次日承接 |
| 周复盘 weekly-review | 是（周度视角） | 中期均线 MA60/120/250 + alpha_20/60d + 板块周度轮动 | 独立，给下周定基调 |

### 4.1 晚间复盘（evening）——全量技术深度

晚间是数据最全、时间最宽裕的时段，是技术深度的主战场。流程：

1. **跑大盘**：对上证/深证成指/创业板指/科创50 四大指数各跑一遍（不取基准、不算相对强弱），得到指数的均线排列、量价、ADX 趋势强度。
2. **跑持仓+自选**：对 `data/positions.json` 每只持仓、`data/watchlist.json` 每只自选股跑引擎（ETF 不取基准，个股取沪深300基准用于相对强弱）。
3. **每只标的产出"四维深度卡片"**：基本面（消息面变化）/ 技术面（状态描述）/ 资金面 / 消息面，末尾加一条**对称条件式情景**和一条**技术位参考**（现价距关键支撑/压力位还有多远）。
4. **大盘 beta vs 个股 alpha**：用 `relative_strength.alpha_20d/60d` 说明持仓今日异动是跟大盘走还是自己走。

卡片写法见第六节模板。这一步替换掉旧报告里"罗列消息 + 打星评分"的浮浅写法。

### 4.2 早间推送（morning）——承接昨晚，不跑引擎

早间时间紧、盘前数据少，**不跑引擎**。技术深度来自两处：

- **隔夜外盘定性**：美股三大指数、欧洲、A50 期货的隔夜方向，定性描述（涨/跌/平、是否破关键位），不硬编点位。
- **今日技术参考位（承接昨晚）**：从昨晚 evening 报告里调出每只持仓的支撑/压力位，作为"今日盘中观察基准"。写成"XX 昨收 A，昨晚测算支撑 B、压力 C——今日若回踩 B 一带可观察是否站稳"。

若昨晚没生成 evening 报告（如周一早晨、节后首日），如实说明"无昨晚参考位"，仅做隔夜外盘定性，不编造位。

### 4.3 盘中复盘（intraday）——技术位互动温度计

盘中是 15 分钟级的快速决策窗口，**不跑引擎**（盘中数据当日 K 线未收，跑出来的指标会失真）。技术深度浓缩成一个"**技术位互动温度计**"：拿盘中现价，去和昨晚 evening 测算的那组位做对照。

对每只持仓输出一行：
- 现价相对昨晚**支撑位**：上方还是下方，距离百分比；
- 现价相对昨晚**压力位**：是否逼近/突破；
- 一句**盘中量价定性**：放量/缩量配合现价方向（盘中量比可用，但 K 线形态不下定论）。

盘中跌破昨晚支撑的持仓，只"标记观察"，不建议卖出（boundaries §1/§7）。

### 4.4 周复盘（weekly）——中期视角，独立跑一次

周复盘把时间尺度从"日"拉到"周"，关注中期趋势而非单日波动。**独立跑一次引擎**，但读数的侧重点不同：

- **中期均线为主**：看 `ma60/120/250` 而非 `ma5/10`。价格站上/跌破年线（MA250）、半年线（MA120）是中期多空的分水岭，比短均线金叉死叉更有意义。
- **相对强弱（alpha）是周复盘的核心**：`relative_strength.alpha_20d/60d` 回答"过去一两个月这只标的相对沪深300是在创造超额收益还是拖后腿"，这是中长期持仓最该关心的维度。正 alpha 且扩大→个股逻辑在兑现；负 alpha 且走阔→需复查基本面叙事是否生变。
- **周度量价**：用 `recent_series` 看本周相对上周的量能变化（周度放量突破 vs 缩量整理）。
- **板块周度轮动**：哪些板块本周走强/走弱，持仓所在板块处在轮动的什么位置。
- **周度条件式情景**：以中期均线为锚，写下周/未来两周的对称情景（"若周线站稳 MA60 则中期趋势修复得到确认 / 若失守 MA120 则需复查持仓基本面是否支撑继续持有"）。

## 五、大盘指数与板块轮动的深度写法

### 5.1 大盘指数（四大指数）

指数（本身就是基准）**不算相对强弱**。深度来自：

- **均线排列 + 位置**：四大指数各自的 `ma_arrangement`（多头/空头/纠缠）和价格相对 MA20/MA60/MA250 的位置，一句话概括各指数处在"健康上行 / 高位震荡 / 趋势走弱 / 底部修复"哪个状态。
- **ADX 定趋势强度**：`adx14`<20 写"震荡市、方向感弱"，>40 写"趋势强劲"，据此给后面的信号"打折"。
- **量价**：用指数 `vol_ratio` 和 `recent_series` 的 volume 描述放量/缩量，配合涨跌方向判断量价是否健康。
- **指数间分化**：上证（权重蓝筹）vs 创业板/科创50（成长）谁强谁弱，揭示市场风格在偏价值还是偏成长。
- **对称情景**：以指数关键均线/整数关口为锚写对称情景，不写"明日涨跌概率"。

### 5.2 板块轮动持续性

板块不跑引擎（引擎面向个股/指数/ETF），板块深度来自复盘脚本的板块涨跌+资金流排名 + 跨日对比：

- **轮动方向**：今日强势板块相对昨日/本周是延续还是切换，资金是在同一主线内扩散还是在不同主线间跳。
- **持续性判断**：领涨板块是"放量+资金净流入+龙头连板"（持续性强）还是"缩量+资金流出+冲高回落"（一日游嫌疑）——用量价资金三者是否共振来定性，不打概率。
- **持仓板块定位**：用户持仓/自选所属板块处在轮动的什么位置（主线/补涨/退潮），决定个股是顺风还是逆风。

## 六、条件式情景与卡片写法模板

### 6.1 持仓四维深度卡片（晚间复盘用）

```
#### 600584 长电科技 ｜ 今日 +X.XX%（板块 +Y%，沪深300 +Z%）
- 技术面：站上 MA5/10/20、MA 多头排列、MACD 在零轴上方金叉延续；
  RSI12=XX（未超买/接近超买）、bias20=+X%（乖离合理/偏大）、ADX=XX（趋势强度中等）。
  → 当前处于「趋势健康上行、短期略有过热」的状态。
- 资金面：主力近X日净流入/流出 …；OBV 20日趋势 上升/走平。
- 消息面：（仅与该股/行业相关的重点，1-2条；无则写"今日无直接相关消息"）
- 相对强弱：alpha_20d=+X%（跑赢/跑输沪深300）→ 今日上涨更多是个股 alpha / 跟随大盘 beta。
- 条件式情景（对称）：若放量站稳 MA20（约 A 元一带）则上行趋势延续得到确认；
  若跌破 MA20 且 MACD 死叉则需警惕转入回调。（A 为参考位，非下单指令）
- 技术位参考：现价 P，下方最近支撑约 XXX（距 P −X%），上方最近压力约 XXX（距 P +X%）。
  均为参考位，非下单指令。
```

要点：技术面落在"状态描述"，情景**两个方向都写**，纪律校准只报"距离 + 是否触及"，绝不写"明日买/卖 XX 元"。

### 6.2 条件式情景句式库（对称，二选一禁止只写一边）

- "若放量站稳 MAxx（约 A）则上行趋势延续得到确认；若缩量跌破则需警惕假突破。"
- "若回踩支撑 B 一带获支撑企稳则中期结构未破；若有效跌破 B 则中期趋势转弱信号。"
- "若 MACD 在零轴上方再度金叉则多头动能恢复；若死叉下穿零轴则动能转弱。"
- "若周线站稳 MA60 则中期趋势修复确认；若失守 MA120 则需复查持仓基本面。"

禁止句式：含"明日大概率"、"上涨概率 XX%"、"建议在 XX 元买入/卖出"、"目标看到 XX 元"的单边预测或下单指令。

## 七、数据不可用时的降级规则

取数按 `data-source-config.json` 路由优先级依次尝试（专业金融 MCP（Wind）→ 本地 Python 行情包 → 联网搜索），任何一环失败都按下列规则降级。**核心红线：宁可缺维度，绝不编数字**。

- **专业金融 MCP（Wind）不可用 / 额度不足**：降级到本地已安装的 Python 行情包取日线 CSV；若本地行情包产出 CSV 成功，后续 `compute_indicators.py` 照常运行，在报告技术面小节注明"本次技术指标由本地脚本计算（Wind MCP 不可用）"。
- **专业 MCP 与本地行情包均失败**：该标的无法获取结构化日线，技术深度整体降级。若仍有联网搜索可用，仅做定性描述（"近期走势偏强/偏弱/震荡"），不给出任何指标数值、均线价位、金叉死叉信号；在报告显著位置写明"技术指标引擎不可用（专业 MCP 及本地行情包均获取失败），本次技术面为定性判断，无指标数值"。
- **某标的取数失败 / 历史不足 60 日**（次新股、停牌）：跳过该标的技术卡片的指标部分，标注"数据不足，技术面跳过"，其余维度正常写。meta 里 `flags.insufficient_history=true` 时主动说明长周期均线不可信。
- **基准指数缺失**（benchmark 抓取失败）：`relative_strength` 为 null，跳过"alpha/相对强弱"维度并说明"本次缺相对大盘强弱"，其余维度照常。
- **取到的 CSV 列名与数据契约不符**：降级为定性描述，不硬编数字。
- **盘中/早间本就不跑引擎**：若需承接的昨晚 evening 报告不存在，如实说明"无昨晚参考位"，只做定性，不回填、不编造历史测算位。

降级时始终遵守 boundaries §7 的"诚实声明技术分析的局限"：缺什么说什么，不用想象的数字填满报告。
