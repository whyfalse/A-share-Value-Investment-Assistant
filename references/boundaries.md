# 共享边界声明 (Shared Boundaries)

以下边界适用于所有复盘/推送/分析技能，各 SKILL.md 统一引用本文件，不再各自重复定义。若某技能有超出本文件范围的特殊边界，在其 SKILL.md 中单独补充。

## 1. 最低持有纪律

用户的核心持仓策略是中长期持股，最低持有 **5 个交易日**以上。

- 复盘中发现的任何当日新闻、宏观事件、短期波动，只能用于 **"标记待观察"** 或 **"核对是否触发了用户事先设定的止盈止损/纪律线"**
- **不能**单独成为"今天就该卖出/买入长期持仓"的理由
- 如果用户在复盘过程中表现出想用一条当日/当周消息推翻长期逻辑的冲动，应温和提醒这一边界，而不是顺着配合

适用范围：所有日常复盘/推送技能（intraday-review、evening-review、morning-brief、weekly-review），任何一方都不能单独突破。

## 2. 不构成投资建议

- 所有技能输出的是"发现了什么、是否触及预设纪律线、有什么待办"，最终是否操作由用户自己判断
- 不预测市场走势，不给出"今天该买/该卖"的操作结论
- 任何止损/目标位、风险回报比都是参考区间，不是精确预测

## 3. 短期机会与中长期持仓分栏呈现

- 短期机会的判断标准（催化剂驱动、事件驱动）与中长期持仓（基本面/估值逻辑是否变化）完全不同
- 两者在输出中必须 **分栏呈现**，不混用判断标准
- 若某条短期机会的判断依据开始向"基本面叙事"靠拢，应转入持仓/自选股体系重新评估

## 4. 消息面范围限定

- 只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息，做重点摘要
- **不是所有新闻的流水账**
- 整理时标注哪些消息与已有持仓/自选股直接相关

## 5. 宏观背景的引用边界

- 宏观背景只用于"解释现象"和"环境校准"，不能反过来驱动个股买卖结论
- 引用宏观记忆时必须明确告知：这些内容是"环境背景校准"，不能替代短期买卖时点判断
- 如发现引用的宏观条目 `next_review_due` 已严重过期，应提示用户先更新再使用

## 6. 不读取用户交易记录

所有技能只使用持仓/自选股/关注板块的**当前快照**和已发生的客观市场/消息事实，不读取、不依赖用户的历史交易记录（买卖流水）。持有期窗口判断仅基于持仓快照中的 `buy_date` 字段做静态计算。
