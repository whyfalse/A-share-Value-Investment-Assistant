# A股晚间复盘(收盘后、当晚)

## 适用边界(Scope) —— 必须遵守

本skill遵守 `../../../references/boundaries.md` 的全部共享边界（最低持有纪律、不构成投资建议、短期/中长期分栏、消息面范围限定、§7技术面深度边界等），以下仅列出本skill特有的边界。**晚间复盘是唯一全量跑技术引擎的时段**：调用方法、JSON字段、四维卡片写法、对称条件式情景、降级规则统一遵循 `../../../references/technical-depth-guide.md`（§4.1 晚间配方），技术读数只用于"客观状态描述 + 纪律线校准"，不预测涨跌、不给买卖价位。

1. **这是"收盘后信息汇总与衔接",不是"盘后操作建议"**:目标是让用户在当晚掌握"今天A股全天发生了什么、同时段亚太市场怎么样、白天有哪些相关消息",并把当日盘中复盘留下的疑点收口——不是据此给出"今晚/明天该买该卖"的结论。
2. **只汇总事实,不预测走势**:当日A股、亚太收盘都是已发生的事实;亚太与A股的联动/背离只客观陈述,不据此预测次日方向。
3. **消息面有明确的时间窗口和范围限定**:白天消息汇总覆盖"今晨早间消息推送之后 → 今晚复盘时"的时段(对应A股盘中+盘后白天这段),与次日 `ashare-morning-brief` 的"隔夜"窗口首尾相接、合起来构成完整24小时、不留缝。范围上只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息,做重点摘要,不是当天所有新闻的流水账。
4. **Tier 3 事件的队列处理**:晚间复盘发现具备 Tier 3(事件驱动)级别的白天事件时,产出更新建议追加至 `../../../data/macro_updates_queue.json`（队列协议详见 `../../../references/tier3-event-handling.md`）,不直接写入 `macro_context.json`。正式条目的新建、candidates 转正、框架整体维护由 `ashare-macro-context` 统一负责。

## 所需本地输入

| 输入 | 内容 | 来源 |
|---|---|---|
| 持仓/自选股记录 | 用于判断白天消息是否与已有标的相关 | `data/positions.json` / `data/watchlist.json` |
| 宏观环境记忆 | 用于判断白天事件是否已被既有背景覆盖,或属于全新信息 | `ashare-macro-context` 维护的本地记忆文档 |
| 当日盘中复盘的未结事项 | 用于在晚间给出结论,避免疑点无限循环 | `output/ashare-intraday-review/intraday_review_[当日].md` |
| 当日A股行情/资金流向 | 用于第1块内容 | 不规定具体获取方式 |
| 亚太市场当日收盘行情 | 用于第2块内容 | 不规定具体获取方式 |
| 白天消息/公告 | 用于第3块内容 | 不规定具体获取方式 |
| 技术指标(全量) | 用于大盘技术面表 + 持仓四维卡片的技术维度 | `ashare-technical-analysis` 引擎,调用方法见 `technical-depth-guide.md` §二 |

若持仓/自选股或宏观记忆文件不存在,提示用户先用对应skill建立(可直接用对应template新建),不要凭空假设内容。若当日盘中复盘存在未结事项而本次仍无法给出结论,应继续保留为"隔夜需观察事项",而不是默默丢弃。

## 复盘内容与执行步骤

### 0. 技术引擎全量运行(盘后第一步,数据落 `output/_tech_cache/`)

晚间是数据最全、最宽裕的时段,先把技术引擎全量跑一遍,产出的均线/支撑压力/纪律距离构成一组"技术参考位",供次日早间/盘中承接。具体调用命令、JSON字段口径见 `../../../references/technical-depth-guide.md`(§二调用方法、§三字段速查、§4.1晚间配方),此处只列要做的事:

1. **大盘四大指数**:对上证(000001)、深证成指(399001)、创业板指(399006)、科创50(000688)各用 `--asset-type index` 跑一遍。
2. **持仓 + 自选股**:对 `positions`/`watchlist` 每只标的跑一遍(ETF 如 159558 用 `--asset-type etf`,个股默认 stock,引擎会自动附沪深300基准算相对强弱)。
3. **降级**:akshare 缺失/某标的取数失败/基准缺失时,严格按 `technical-depth-guide.md` §七降级——缺哪个维度说哪个,**绝不编造指标数字**填报告。
4. 技术读数只用于"客观状态描述 + 纪律线校准",写法见下方第1块、第5块,边界见 `boundaries.md` §7。

### 1. A股当日全天复盘(优先级最高)
- 主要指数收盘表现(上证、深证、创业板、北证等)及涨跌幅
- 成交额、北向/南向资金当日净流向(只需当日数据本身,中长期趋势交给宏观记忆)
- 领涨/领跌板块、涨跌停家数等市场情绪指标
- **大盘技术面(承接第0步指数引擎结果)**:四大指数各自的均线排列(多头/空头/纠缠)、价格相对 MA20/MA60/MA250 的位置、ADX 趋势强度(<20震荡、>40强趋势)、量价是否健康;指数间分化(权重蓝筹 vs 成长)揭示市场风格偏向。一句话概括每个指数处在"健康上行/高位震荡/趋势走弱/底部修复"哪个状态,并给一条以关键均线/整数关口为锚的**对称条件式情景**(写法见 `technical-depth-guide.md` §5.1),不写涨跌概率。
- 交叉核对持仓/自选股记录:今日是否有相关标的公告或异动。若 `ashare-intraday-review` 当天已记录,这里只做衔接提示,不重复展开

### 2. 亚太市场辅助
- 日经、恒指、韩股(KOSPI)等当日收盘表现——与A股处于同一交易时段,当晚已是完整收盘数据
- 与A股当日的联动或背离,仅客观陈述,不据此预测次日方向(海外完整的隔夜行情由次日早间消息推送处理)

### 3. 白天消息面整理
- **时间窗口**:覆盖"今晨早间消息推送之后 → 今晚复盘时",承接早间消息推送的隔夜窗口,避免两次复盘之间出现信息盲区
- **范围限定**:只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息,客观陈述,不做解读延伸,不是当日所有新闻的流水账
- **分类整理**(便于快速扫读,也便于与 `ashare-macro-context` 维度框架对应):按"国内政策与监管"/"行业与产业动态"/"公司层面"/"国际重大事件(白天如有)"分组摘要,而不是按时间顺序罗列 (完整分类标准见 `../../../references/message-classification.md`)
- **与持仓/自选股的关联标注**:整理时标注哪些消息与已有持仓/自选股直接相关,作为"隔夜需观察事项"的来源依据之一
- 判断是否有 Tier 3 级别的新宏观事件,产出更新建议追加至 `data/macro_updates_queue.json`（队列协议详见 `../../../references/tier3-event-handling.md`）,并注明建议的类别代码(如 A_政治与政策、J_资本市场制度政策等)

### 4. 板块轮动持续性(承接领涨领跌排名)
- **轮动方向**:今日强势板块相对昨日/本周是延续还是切换,资金是在同一主线内扩散还是在不同主线间跳
- **持续性定性**:领涨板块是"放量+资金净流入+龙头连板"(持续性强)还是"缩量+资金流出+冲高回落"(一日游嫌疑)——用量价资金三者是否共振来定性,**不打概率**
- **持仓板块定位**:用户持仓/自选所属板块处在轮动的什么位置(主线/补涨/退潮),决定个股是顺风还是逆风(写法见 `technical-depth-guide.md` §5.2)

### 5. 持仓/自选股四维深度卡片(替换旧版"罗列消息+打星")
对每只持仓/自选股产出一张"四维深度卡片",这是晚间复盘技术深度的主战场,**短期机会与中长期持仓分栏呈现,口径不混用**(boundaries §3):
- **技术面**:用第0步引擎结果做"客观状态描述"——均线排列、MACD/KDJ/RSI 位置、bias20 乖离、ADX 趋势强度,翻译成"当前处在什么技术状态"(如"趋势健康上行、短期略过热"),不甩裸数字
- **资金面**:主力近期净流入/流出、OBV 趋势
- **消息面**:仅与该股/行业相关的重点(无则写"今日无直接相关消息"),不做泛新闻流水账
- **相对强弱(alpha vs beta)**:用 `relative_strength.alpha_20d/60d` 说明今日异动是个股 alpha 还是跟随大盘 beta
- **对称条件式情景**:一条"若…则…/若…则…"两个方向都列出的情景(以参考位为锚,标注"参考位非下单指令")
- **纪律线校准**:现价距用户预设止盈/止损/加仓线还有多远、是否触及;未满最低持有5日的持仓即便出现空头信号也只"标记观察/核对止损线",不建议提前卖出(boundaries §1/§7)

卡片完整写法与句式见 `technical-depth-guide.md` §6.1/§6.2。

### 6. 衔接与承接
- **核对当日盘中复盘未结事项**:逐条核对盘中复盘"待办/留待晚间复盘的疑点",凡当晚已有新信息的给出结论(已解释清楚/仍在观察/已转入宏观记忆),不让同一事项无限循环却始终没有结论
- **产出隔夜需观察事项**:把今晚需要跟进隔夜进展的标的/主题列出,交棒给次日 `ashare-morning-brief`(它会核对这些事项的隔夜进展);**一并交棒第0步算出的技术参考位**(各持仓的支撑/压力/纪律线),供次日早间/盘中承接
- **中长期持仓如遇白天重大消息**:明确写"已标记观察,按既有止盈止损/纪律线处理,不建议盘后临时操作",而不是给出操作倾向

## 输出格式

按以下结构输出,保持简洁,每条一两句话:

```
## 晚间复盘 [日期]

### 今日A股全天回顾
- 指数表现 / 资金流向 / 领涨领跌板块 / 涨跌停情绪
- 大盘技术面:四大指数均线排列 / 相对MA20/60/250位置 / ADX趋势强度 / 量价 / 风格分化
  → 每个指数处在「健康上行/高位震荡/趋势走弱/底部修复」哪个状态 + 一条对称条件式情景
- 持仓与自选股相关提示(如有,与盘中复盘衔接不重复)

### 亚太市场(辅助)
- 日经/恒指/韩股收盘表现
- 与A股的联动/背离(仅客观陈述)

### 白天消息面(今晨早间消息推送→今晚)
- 国内政策与监管:[摘要]
- 行业与产业动态:[摘要]
- 公司层面:[摘要,标注与持仓/自选股的关联]
- 国际重大事件:[白天如有]

### 板块轮动持续性
- 轮动方向(延续/切换) / 领涨板块持续性定性(量价资金是否共振) / 持仓板块在轮动中的位置

### 持仓/自选股四维深度卡片(逐只)
#### [代码 名称] ｜ 今日 ±X.XX%(板块 ±Y%,沪深300 ±Z%)
- 技术面:[均线排列/MACD/KDJ/RSI/bias/ADX → 客观状态描述]
- 资金面:[主力净流入流出 / OBV趋势]
- 消息面:[仅相关重点,无则"今日无直接相关消息"]
- 相对强弱:alpha_20d=±X% → 个股alpha / 大盘beta
- 条件式情景(对称):若…则…;若…则…(参考位非下单指令)
- 纪律校准:现价距止盈/止损/加仓线 ±X%,是否触及(未满5日持仓只标记观察)

### 短期机会发掘
- [标的]: 今日全天观察到的短期催化剂/异动 → 为什么值得注意
- (若无)今日未发掘到值得注意的短期机会

### 盘中未结事项核对
- [事项]: 结论(已解释/仍观察/转入宏观记忆)

### 已完成宏观更新队列
- (如有Tier3新事件)已产出更新建议至队列:[类别代码 + 维度名称 + 更新内容简述]
- (若无)今日无需产出宏观更新建议的新事件

### 隔夜需观察事项(交棒明早早间消息推送)
- [标的/主题]: 关注原因,仅作观察标记,非操作建议
- 技术参考位移交:各持仓支撑/压力/纪律线(供次日早间/盘中承接)
```

## 输出保存

完成上述复盘后,除了在对话中展示,还必须把完整报告落盘保存到 `../../../output/ashare-evening-review/` 文件夹下,文件名为 `evening_review_[日期].md`(日期用当日,格式 `YYYY-MM-DD`,如 `evening_review_2026-06-25.md`)。若文件夹不存在则先创建。同名文件已存在时直接覆盖(同一天重复复盘以最新一次为准)。

## 与其他skill的关系

- **`ashare-intraday-review`**:本skill承接其当日盘中复盘留下的"待办/未结疑点"并尽量给出结论,形成"盘中→晚间"的收口;两者共享同一套最低持有纪律。短期机会方面,盘中复盘和晚间复盘各自从不同的时间窗口独立发掘新机会。
- **`ashare-morning-brief`**:本skill产出的"隔夜需观察事项"交棒给次日早间消息推送跟进隔夜进展;白天消息窗口与早间消息推送的隔夜窗口首尾相接,合成完整24小时闭环。短期机会方面,晚间复盘和早间推送各自从不同的时间窗口独立发掘新机会。
- **`ashare-macro-context`**:本skill对 Tier 3 级新事件产出更新建议追加至队列,由 macro-context 统一消费处理。正式条目的新建、candidates 转正、框架级维护仍由该skill负责。
- **`ashare-technical-analysis`**:晚间复盘的第0步会直接调用该引擎的脚本(`fetch_data.py` + `compute_indicators.py`)对四大指数和每只持仓/自选股全量算技术指标,作为大盘技术面表和持仓四维卡片的数据来源——技术深度已内置,不再外包。调用方法/字段口径/降级规则统一遵循 `../../../references/technical-depth-guide.md`。若用户需要对某只标的做更长篇的纯技术面专题报告(完整的多维指标解读+图表),仍可另行单独触发该skill展开。
