# A股晚间复盘(收盘后、当晚)

## 适用边界(Scope) —— 必须遵守

本skill遵守 `../../../references/boundaries.md` 的全部共享边界（最低持有纪律、不构成投资建议、短期/中长期分栏、消息面范围限定等），以下仅列出本skill特有的边界：

1. **这是"收盘后信息汇总与衔接",不是"盘后操作建议"**:目标是让用户在当晚掌握"今天A股全天发生了什么、同时段亚太市场怎么样、白天有哪些相关消息",并把当日盘中复盘留下的疑点收口——不是据此给出"今晚/明天该买该卖"的结论。
2. **只汇总事实,不预测走势**:当日A股、亚太收盘都是已发生的事实;亚太与A股的联动/背离只客观陈述,不据此预测次日方向。
3. **消息面有明确的时间窗口和范围限定**:白天消息汇总覆盖"今晨早间消息推送之后 → 今晚复盘时"的时段(对应A股盘中+盘后白天这段),与次日 `ashare-morning-brief` 的"隔夜"窗口首尾相接、合起来构成完整24小时、不留缝。范围上只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息,做重点摘要,不是当天所有新闻的流水账。
4. **Tier 3 事件的队列处理**:晚间复盘发现具备 Tier 3(事件驱动)级别的白天事件时,产出更新建议追加至 `../../../data/macro_updates_queue.json`（队列协议详见 `../../../references/tier3-event-handling.md`）,不直接写入 `macro_context.json`。正式条目的新建、candidates 转正、框架整体维护由 `ashare-macro-context` 统一负责。

## 所需本地输入

| 输入 | 内容 | 来源 |
|---|---|---|
| 持仓/自选股记录 | 用于判断白天消息是否与已有标的相关 | `ashare-intraday-review` 维护的 `positions`/`watchlist` 本地文件 |

| 宏观环境记忆 | 用于判断白天事件是否已被既有背景覆盖,或属于全新信息 | `ashare-macro-context` 维护的本地记忆文档 |
| 当日盘中复盘的未结事项 | 用于在晚间给出结论,避免疑点无限循环 | `ashare-intraday-review` 当日输出的"待办/留待晚间复盘"清单 |
| 当日A股行情/资金流向 | 用于第1块内容 | 不规定具体获取方式 |
| 亚太市场当日收盘行情 | 用于第2块内容 | 不规定具体获取方式 |
| 白天消息/公告 | 用于第3块内容 | 不规定具体获取方式 |

若持仓/自选股或宏观记忆文件不存在,提示用户先用对应skill建立(可直接用对应template新建),不要凭空假设内容。若当日盘中复盘存在未结事项而本次仍无法给出结论,应继续保留为"隔夜需观察事项",而不是默默丢弃。

## 四块复盘内容

### 1. A股当日全天复盘(优先级最高)
- 主要指数收盘表现(上证、深证、创业板、北证等)及涨跌幅
- 成交额、北向/南向资金当日净流向(只需当日数据本身,中长期趋势交给宏观记忆)
- 领涨/领跌板块、涨跌停家数等市场情绪指标
- 交叉核对持仓/自选股记录:今日是否有相关标的公告或异动。若 `ashare-intraday-review` 当天已记录,这里只做衔接提示,不重复展开

### 2. 亚太市场辅助
- 日经、恒指、韩股(KOSPI)等当日收盘表现——与A股处于同一交易时段,当晚已是完整收盘数据
- 与A股当日的联动或背离,仅客观陈述,不据此预测次日方向(海外完整的隔夜行情由次日早间消息推送处理)

### 3. 白天消息面整理
- **时间窗口**:覆盖"今晨早间消息推送之后 → 今晚复盘时",承接早间消息推送的隔夜窗口,避免两次复盘之间出现信息盲区
- **范围限定**:只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息,客观陈述,不做解读延伸,不是当日所有新闻的流水账
- **分类整理**(便于快速扫读,也便于与 `ashare-macro-context` 维度框架对应):按"国内政策与监管"/"行业与产业动态"/"公司层面"/"国际重大事件(白天如有)"分组摘要,而不是按时间顺序罗列 (完整分类标准见 `../../../references/message-classification.md`)
- **与持仓/自选股的关联标注**:整理时标注哪些消息与已有持仓/自选股直接相关,作为"隔夜需观察事项"的来源依据之一
- 判断是否有 Tier 3 级别的新宏观事件,产出更新建议追加至 `data/macro_updates_queue.json`（队列协议详见 `../../../references/tier3-event-handling.md`）,并注明建议的类别代码(如 A_政治与政策、J_资本市场制度政策等)

### 4. 衔接与承接
- **核对当日盘中复盘未结事项**:逐条核对盘中复盘"待办/留待晚间复盘的疑点",凡当晚已有新信息的给出结论(已解释清楚/仍在观察/已转入宏观记忆),不让同一事项无限循环却始终没有结论
- **产出隔夜需观察事项**:把今晚需要跟进隔夜进展的标的/主题列出,交棒给次日 `ashare-morning-brief`(它会核对这些事项的隔夜进展)
- **中长期持仓如遇白天重大消息**:明确写"已标记观察,按既有止盈止损/纪律线处理,不建议盘后临时操作",而不是给出操作倾向

## 输出格式

按以下结构输出(模板见 `assets/evening_review_template.md`),保持简洁,每条一两句话:

```
## 晚间复盘 [日期]

### 今日A股全天回顾
- 指数表现 / 资金流向 / 领涨领跌板块 / 涨跌停情绪
- 持仓与自选股相关提示(如有,与盘中复盘衔接不重复)

### 亚太市场(辅助)
- 日经/恒指/韩股收盘表现
- 与A股的联动/背离(仅客观陈述)

### 白天消息面(今晨早间消息推送→今晚)
- 国内政策与监管:[摘要]
- 行业与产业动态:[摘要]
- 公司层面:[摘要,标注与持仓/自选股的关联]
- 国际重大事件:[白天如有]

### 短期机会发掘
- [标的]: 今日全天观察到的短期催化剂/异动 → 为什么值得注意
- (若无)今日未发掘到值得注意的短期机会

### 盘中未结事项核对
- [事项]: 结论(已解释/仍观察/转入宏观记忆)

### 已完成宏观更新队列
- (如有Tier3新事件)已产出更新建议至队列:[类别代码 + 维度名称 + 更新内容简述]
- (若无)今日无需产出宏观更新建议的新事件

### 隔夜需观察事项(交棒明早早间消息推送)
- [标的/主题]: 关注原因,仅作观察标记,非操作建议
```

## 与其他skill的关系

- **`ashare-intraday-review`**:本skill承接其当日盘中复盘留下的"待办/未结疑点"并尽量给出结论,形成"盘中→晚间"的收口;两者共享同一套最低持有纪律。短期机会方面,盘中复盘和晚间复盘各自从不同的时间窗口独立发掘新机会。
- **`ashare-morning-brief`**:本skill产出的"隔夜需观察事项"交棒给次日早间消息推送跟进隔夜进展;白天消息窗口与早间消息推送的隔夜窗口首尾相接,合成完整24小时闭环。短期机会方面,晚间复盘和早间推送各自从不同的时间窗口独立发掘新机会。
- **`ashare-macro-context`**:本skill对 Tier 3 级新事件产出更新建议追加至队列,由 macro-context 统一消费处理。正式条目的新建、candidates 转正、框架级维护仍由该skill负责。
- **`ashare-technical-analysis`**:若晚间复盘发现某持仓/自选股需要更深入的技术面研判,提示用户另行触发该skill,不在本次复盘内展开。
