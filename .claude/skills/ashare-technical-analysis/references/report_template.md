# 输出报告模板

生成最终报告时,使用下面这个结构。【】中是需要你结合JSON输出与自己的
判断填写的内容,不是要逐字照抄——尤其是"形态描述"部分,
应该用你自己的语言把均线/量价的走势讲清楚,而不是只甩数字。

如果某个数据点缺失(比如某个维度因数据不足被跳过),如实说明缺失,
不要编造数字填充。

---

```markdown
# 【股票名称/代码】技术面分析报告(中长期)

> 数据区间:【date_range[0]】 至 【date_range[1]】 | 数据来源:【data_source_used】
> 分析日期:【今天日期】 | 最新收盘价:【latest.close】(【latest.pct_chg】%)

【如果risk_overrides非空,在最开头用一段加粗文字突出风险提示,例如:
**⚠ 注意:本次分析触发以下数据质量/风险提示,结论置信度已下调,请谨慎参考——
[列出risk_overrides里的内容]**】

## 一、综合结论

- **结论**:【verdict】
- **置信度**:【confidence】(高/中/低)
- **加权技术评分**:【total_score】(范围-2到+2,正值偏多/负值偏空/接近0为中性)
- **建议适用周期**:中长期,非日内/超短线信号

【用1-2句话总结"为什么是这个结论"——挑出2-3个权重最高、最关键的支持/反对理由,
不要把5个维度的所有reasons都堆在这里,那是下面分维度部分的内容。】

## 二、趋势分析(权重最高的维度)

【结合signals.ma_arrangement、price_vs_ma250、adx14等,
用自己的话描述当前均线排列形态、价格相对年线位置、ADX显示的趋势强弱。
这部分要体现"看图说话"的能力,而不是单纯复述JSON字段。】

- 趋势维度得分:【dimension_scores.trend.score】(权重【weight】)
- 依据:【列出reasons】

## 三、动量与超买超卖

【描述RSI/KDJ当前所处区间、MACD金叉死叉或多空位置、是否存在背离信号。
如果存在顶背离/底背离,这是相对重要的预警信号,值得单独强调。】

- 动量维度得分:【dimension_scores.momentum.score】(权重【weight】)
- 依据:【列出reasons】

## 四、量能分析

【描述最新量比、放量/缩量与涨跌的配合关系、OBV趋势方向,
判断当前走势是否有真实资金支持。】

- 量能维度得分:【dimension_scores.volume.score】(权重【weight】)
- 依据:【列出reasons】

## 五、波动率与当前位置

【描述布林带%b、乖离率,判断当前是"追高风险大"还是"超跌可能存在修复机会",
注意区分"对已持有者"和"对新进场者"的含义可能不同——
比如强趋势+略超买,对持有者是继续持有的理由,对新进场者则要警惕追高。】

- 波动位置维度得分:【dimension_scores.volatility_position.score】(权重【weight】)
- 依据:【列出reasons】

## 六、相对大盘强弱

【如果该维度因基准数据缺失被跳过,说明"本次未纳入相对强弱比较,
权重已分配至其他维度",不要假装有数据。
如果有数据,说明近20/60日跑赢或跑输基准指数的幅度。】

- 相对强弱维度得分:【dimension_scores.relative_strength.score 或 "未纳入"】

## 七、关键支撑/阻力位

| 类型 | 价位 | 备注 |
|---|---|---|
| 阻力位1 | 【resistance[0].level】| 【touches次数/备注】|
| 阻力位2 | 【resistance[1].level】| |
| 当前价 | 【latest.close】| |
| 支撑位1 | 【support[0].level】| |
| 支撑位2 | 【support[1].level】| |

## 八、操作建议(仅供参考,不构成投资建议)

- **建议方向**:【根据verdict转述,如"可考虑逢低分批介入" / "建议观望" / "建议逢高减仓"】
- **止损参考**:支撑位止损约【stop_loss_candidates.support_based】,
  或按2倍ATR计算约【stop_loss_candidates.atr_based_2x】
  (两者权衡:支撑位止损更贴合价格结构但空间可能较大;ATR止损更贴合
  正常波动幅度,二者可结合自己的风险承受能力选择,或取两者间的位置)
- **目标位参考**:阻力位约【target_candidates.resistance_based】,
  或按3倍ATR计算约【target_candidates.atr_based_3x】
- **风险回报比(示意)**:约【risk_reward_ratio_illustrative】
  (这是用上述某一组止损/目标位计算的示意值,不是精确预测,
  仅供判断"值不值得承担这个风险"参考)
- **持仓周期提示**:【holding_period_note,可适当改写但不要丢失核心意思】

## 九、风险提示与免责声明

- 本报告仅基于历史价格与成交量的技术面分析生成,**不涉及公司基本面、
  财务状况、行业前景、新闻舆情等信息**,技术指标具有滞后性和局部失真的可能。
- 技术分析无法精确预测未来涨跌,历史规律不代表未来一定重演,
  本报告中的止损/目标位均为参考性区间,不是精确预测值。
- **本报告不构成证券投资建议**,不代表任何买卖指令,
  最终投资决策及其风险由使用者自行承担,请结合自身风险承受能力、
  资金状况及其他信息来源综合判断,必要时咨询持牌投资顾问。
- A股市场存在政策、流动性、个股特殊事件(停牌、ST、退市等)等
  技术分析无法覆盖的风险,请勿仅凭本报告进行重大投资决策。
```

## 撰写报告时的几条原则

1. **诚实优先于"好用"**。如果指标互相矛盾、数据质量存在问题,
   报告应该如实反映这种不确定性,而不是为了给出一个干脆的结论而
   选择性忽略矛盾信号。
2. **解释"为什么",不要只甩数字**。同样是"RSI=75",在强趋势多头排列中
   和在已经滞涨、量能萎缩的背景下,含义完全不同——这种结合上下文的
   判断是模型该做的事,score.py只负责给出原始分数和触发原因列表。
3. **每次都要带免责声明**,不要因为用户多次使用而省略,
   也不要在用户表现出强烈情绪化交易冲动时弱化风险提示
   (比如用户说"我都重仓了,你就告诉我能不能涨"——
   仍应给出客观技术面信息和风险提示,而不是迎合情绪给确定性承诺)。
4. **不要编造未提供的信息**。如果脚本输出中没有某个数据
   (比如换手率、相对强弱),报告中如实说明"暂无该数据",
   不要为了让报告看起来完整而编造数字。
