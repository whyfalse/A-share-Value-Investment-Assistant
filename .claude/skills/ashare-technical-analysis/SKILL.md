# A股技术面分析(中长期)

本skill遵守 `../../../references/boundaries.md` 中的共享边界（不构成投资建议、最终决策与风险由用户自行承担等）。

## 这个skill做什么 / 不做什么

**做**:基于历史日线价格与成交量,计算趋势、动量、量能、波动率/位置、
相对大盘强弱五个维度的技术指标,综合给出适用于**中长期**
持仓决策的买卖建议,并附止损/目标位参考。

**不做,也不应该被诱导去做**:
- 不分析财务报表、估值水平、行业景气度等基本面信息
- 不分析新闻、研报、社交媒体情绪等消息面信息
- 不提供日内交易、T+0、超短线信号——如果用户明确
  是要做超短线,应如实告知本skill的指标参数(年线、季线等)是为中长期
  设计的,对超短线场景参考价值有限,而不是直接套用结论
- 不支持港股、美股、期货期权、可转债、基金——如果用户提供的代码/名称
  不是A股,应明确说明本skill的局限并停止套用其结论
- 不做"精确预测未来涨多少/跌多少"——技术分析只能给方向性、区间性的
  参考,任何看起来过于精确的数字预测都应该被改写为区间或定性判断

## 工作流程

### 步骤0:确认输入

把用户给的代码/名称标准化为标准A股代码(6位数字 + 交易所,如
`600519.SH`、`000001.SZ`)。如果用户只给了股票名称没给代码,
先尝试用你已有的知识或网络搜索确认代码,代码不确定时跟用户确认一下,
避免分析错股票。如果识别出这不是A股(港股代码、美股代码、基金代码等),
直接说明本skill不支持,不要勉强套用。

### 步骤1:获取数据

这一步的目标只有一个:拿到一份**标准化的日线CSV**,供步骤2消费。

**使用本地已安装的行情工具取数**。根据 `data-source-config.json` 中的路由配置,
优先使用专业金融MCP工具获取A股K线日线数据;如不可用或额度不足,
则改用本地已安装的 Python 行情包;都不可用时走联网搜索兜底
(仅采信定性信息,精确数字注明未核对)。

**无论用哪个工具,产出必须满足下面的数据契约**,否则步骤2无法计算:

- 文件:一份 CSV(默认放 `./output/<code6>_daily.csv`)
- 必需列:`date`(YYYY-MM-DD)、`open`、`high`、`low`、`close`、`volume`
- 可选列:`amount`、`turnover`、`pct_chg`(缺失时步骤2/3会自行处理或降级)
- 价格口径:**前复权(qfq)**,保证均线/指标在除权除息后连续可比;
  若数据源只能给不复权数据,需在报告里说明这一局限
- 排序:按 `date` 升序、无重复日期;回溯长度尽量≥250个交易日(约2年),
  以保证年线等长周期指标有效
- 标的范围:个股给前复权日线;指数/ETF同样落成上述列(指数无复权)。
  注意 `000001` 这类代码既可能是指数也可能是个股,取数时务必确认取到的标的正确

取数后需自行做数据质量审视:
- 检查历史数据是否≥60个交易日(≥250更佳)
- 是否 ST/*ST
- 是否有长期停牌缺口(连续无交易天数异常)
- 近期是否频繁涨跌停

将质量检查结果构造为一份等价 meta(至少含 `rows`、`date_range`、`flags`),
在步骤3传给 `score.py`。**不要在没���数据的情况下编造分析结论**。

### 步骤2:计算技术指标(`scripts/compute_indicators.py`)

```bash
python scripts/compute_indicators.py --csv ./output/600519_daily.csv \
  [--benchmark-csv ./output/benchmark_000300.csv] \
  --out-json ./output/600519_indicators.json
```

- `--benchmark-csv` 传入基准指数(默认沪深300)的CSV——只需 `date`、
  `close` 两列——即可启用相对强弱维度;没有则跳过该维度。
- 输出包含`latest`(最新指标值)、`signals`(金叉死叉/背离等离散信号)、
  `key_levels`(支撑阻力位)、`recent_series`(近期走势精简序列)。
  各指标的计算口径和参数选择理由见 `references/indicators_guide.md`,
  通常不需要每次都读,遇到数字看不懂或想验证时再查。

### 步骤3:多维度打分(`scripts/score.py`)

```bash
python scripts/score.py --indicators-json ./output/600519_indicators.json \
  --meta-json ./output/600519_meta.json --out-json ./output/600519_score.json
```

- 输出五个维度的分数与理由、加权总分、结论文字、置信度、
  止损/目标位候选、风险否决信息。完整的打分规则、权重、分档标准、
  风险否决条件见 `references/scoring_framework.md`。
- 这一步已经把"要不要否决结论改为观望"这种风险判断做掉了,
  你不需要重新推导,但要在报告里把`risk_overrides`里的内容讲清楚
  ——用户需要知道"为什么这次置信度是低,即使分数看起来不小"。

### 步骤4:撰写最终报告

用步骤1-3的JSON输出,按照 `references/report_template.md` 里的
固定结构撰写中文Markdown报告。这一步**不是简单地把JSON字段抄进模板**——
你需要:
- 对趋势形态、量价关系等做出有判断力的文字描述(比如"均线尚未形成
  清晰的多头排列,但MA5已上穿MA20,如果后续放量突破MA60可能确认
  趋势反转"),而不是只甩"trend_score=0.5"这种数字
- 当五个维度互相矛盾时,明确指出矛盾本身,而不是挑顺眼的维度强调、
  弱化不顺眼的维度
- 任何止损/目标位、风险回报比都要说明这是参考区间,不是精确预测
- 报告结尾必须包含风险提示与免责声明(模板里已经写好了标准措辞,
  可以适当改写但不要删减核心要点:不构成投资建议、技术分析的局限性、
  最终决策与风险由用户自行承担)

撰写完成后,除了在对话中展示,还必须把完整报告落盘保存到
`../../../output/ashare-technical-analysis/` 文件夹下,文件名为
`<标准代码>_technical_[日期].md`(代码用步骤0标准化后的形式、去掉点号,
日期格式 `YYYY-MM-DD`,如 `600519SH_technical_2026-06-25.md`)。
若文件夹不存在则先创建,同名文件已存在时直接覆盖。注意这里保存的是
最终中文报告,与步骤1-3脚本输出到 `./output/` 的 CSV/JSON 中间产物分开。

## 贯穿全程的四条原则

- **中长期视角**:本skill的指标参数(20/60/120/250日均线)为中长期持仓决策设计。均线、ADX、量能互相打架时大概率处于趋势转换模糊期——"如实承认看不清"比"硬给方向"更有价值。
- **诚实输出不确定性**:总分中性或置信度低时直接建议观望,不因用户想要"买/卖"答案而倒向一边。
- **风险优先**:报告重点放在"风险回报比"和"止损位空间",不渲染目标涨幅。用户有情绪化倾向时给出客观信息和风险提示,不强化冲动。
- **不编造数据**:脚本输出缺失的字段如实说明,不伪造未实际算出的指标(见 `../../../references/boundaries.md` §7)。

## 引用文件

- `scripts/compute_indicators.py`——计算技术指标、支撑阻力位、金叉死叉、背离信号
- `scripts/score.py`——五维度加权打分、结论生成、止损止盈参考
- `references/indicators_guide.md`——各指标计算口径与解读(排查"这数字怎么来的"时查)

- `references/report_template.md`——最终报告结构模板(撰写步骤4时必读)
