# A股复盘数据看板(移动端HTML渲染)

## 适用边界(Scope)

本skill遵守 `../../../references/boundaries.md` 的全部共享边界,正文不再逐条复述。以下仅列本skill特有边界:

1. **这是"展示层",不是"分析层"**:本skill把用户引用的已有报告/数据内容渲染成移动端HTML看板。它不取实时行情、不跑技术引擎、不做新的买卖推导——看板里出现的每一项结论与数字,都必须能在用户引用的数据源内容里找到出处,绝不凭空生成或外推。
2. **数据缺失按模板规则处理,不编造**:数据源里缺失的字段(如某指数成交额为"—"、某指标未算出),严格按模板资源里规定的"数据缺失处理逻辑"渲染(占位符 / 灰显 / "数据缺失"标注 / 整块降级等),不得用估算值或编造数字填满。这与 `boundaries.md` §7"不编造数字"一致。
3. **忠实呈现,不二次解读**:看板是数据源内容的可视化复刻,不改变其口径与结论。原文里"对称条件式情景"仍双向列出、参考位仍标注"非下单指令"、短期机会与中长期持仓仍分栏——这些纪律在渲染时原样保留,不得为了"好看"改成单边预测或合并口径(§1/§2/§3/§7)。
4. **移动端优先**:输出HTML针对手机竖屏阅读优化——viewport、单栏/窄栏布局、可触摸点击的字号与间距、深色/高对比配色、首屏即核心结论。可在大屏自适应展开,但不以PC宽屏为首要目标。
5. **不预设报告类型、不内置映射表**:本skill默认不知道有哪些报告类型,也不维护"报告类型→数据源路径/模板文件"的映射。用哪个模板、数据源是什么,完全由"用户提示词 + 提示词引用的内容 + `templates/` 下已存在的模板"三者动态结合来判断(见下文工作流)。`templates/` 为空或无匹配模板时,提示用户放置/创建模板,不要硬套不相关模板或凭空自创架构。

## 工作流程

### 步骤0:理解用户提示词与引用内容
从提示词中提取:
- 用户想渲染的对象(描述了哪份报告/哪类数据);
- 是否显式引用了数据源——一个文件路径、一段粘贴的报告正文,或仅口头描述;
- 是否点名了某个模板(如"用 XX 模板")。

若提示词未引用任何具体内容、且无法据此定位数据源,向用户确认:请提供报告文件路径或粘贴报告正文。**不要凭"报告类型"猜一个写死的路径**。

### 步骤1:扫描并读取已存在的模板
列出 `templates/` 下所有 `*.html`(忽略 README 等非模板文件),逐个读取其**自描述**——模板应在文件首部用 HTML 注释声明自己面向什么数据/报告(自描述格式见 `templates/README.md`);无自描述时退而读取其 `<title>`、模块注释与整体结构来推断用途。据此建立"可用模板 → 各自面向什么"的清单。

- `templates/` 为空或无任何模板 → 如实告知用户尚未放置模板,提示按 `templates/README.md` 契约创建,停止。

### 步骤2:匹配模板
用步骤0的"提示词 + 引用内容"与步骤1的"可用模板自描述"做匹配,选出最贴合的模板:
- 用户显式点名某模板 → 直接用该模板(若不存在则告知)。
- 未点名 → 按自描述的匹配关键词 / 面向数据,选与引用内容最相符的模板;多个同样贴切时向用户确认,不要替用户拍板。
- 无任何模板能合理匹配 → 告知用户当前 `templates/` 下无合适模板,提示创建,不要硬套一个不相关的模板。

### 步骤3:定位并读取数据源
数据源来自用户引用的内容,不查写死的路径表:
- 用户给了文件路径 → 读取该文件(Markdown 报告 / JSON 等),完整通读,梳理可渲染单元(章节、表格、卡片、情景、消息分组、观察清单等)及顶部"数据源说明"备注(标注哪些是精确值、哪些是定性/未核对)。
- 用户粘贴了正文 → 以粘贴内容为数据源,同样梳理可渲染单元。
- 用户仅口头描述、未给路径或正文 → 向用户索要文件路径或正文,不要假设路径。
- 引用的文件不存在或内容为空 → 如实告知,停止。

### 步骤4:数据绑定(数据源 → 模板模块)
把步骤3梳理出的数据,按步骤2模板的模块描述逐一绑定:每个模块对应数据源里的哪个章节/表格/字段,以模板规定的展示形式落地。绑定过程遵守:
- **只搬不造**:只填数据源里真实存在的数据;没有的字段触发该模块的"数据缺失处理逻辑",不外推、不估算、不抄别处数字顶替。
- **口径不变**:涨跌方向、百分比、参考位标注("非下单指令")、分栏(短期机会 vs 中长期持仓)、对称情景双向等,与原文一致,不因排版改动口径。
- **数字精度**:沿用原文精度,不做四舍五入放大或伪精确(原文写"约6,601亿"就照搬,不改成"6,601.00亿")。

### 步骤5:生成移动端HTML看板
以模板的架构与样式为蓝本,把步骤4绑定的数据渲染成一份独立的单文件HTML:
- **移动端优先**:`<meta name="viewport" content="width=device-width, initial-scale=1.0">`;主体单栏、最大宽度受限(如 ≤480–560px 居中),手机竖屏首屏即见核心结论;字号与点击区间满足触摸操作;深色高对比配色便于夜间阅读。
- **自包含**:CSS内联、不依赖本地额外资源(可引用 Google Fonts 等公共CDN字体,与模板保持一致);图标优先用内联SVG或emoji,保证单文件可离线打开。
- **缺失处理落地**:凡数据源缺失的字段,按模板"数据缺失处理逻辑"渲染占位(如"—"、"数据缺失"灰显、或该卡片标注"本次未提供"),绝不在HTML里塞入数据源里没有的数字。
- **顶部数据源说明**:把数据源里的"数据源说明"备注以小字呈现在看板顶部或底部,如实标注哪些是精确值、哪些是定性/未核对,与原文一致。
- **保留纪律措辞**:参考位"非下单指令"、对称情景双向、风险提示与免责声明等,在对应模块原样保留,不得为版面精简而删除。

### 步骤6:输出保存
完成渲染后,把HTML单文件落盘保存到 `../../../output/ashare-dashboard/` 文件夹下,文件名由"匹配到的模板"派生:`<模板文件名主干>_dashboard_[日期].html`(模板文件名主干即匹配到的模板去掉 `.html` 后缀;日期取自数据源内容或用户指定,格式 `YYYY-MM-DD`)。文件夹不存在则先创建,同名文件已存在时直接覆盖(同一天重复生成以最新一次为准)。除落盘外,在对话中给出文件路径,并可贴出看板首屏结构示意供用户确认。

## 贯穿全程的原则

- **展示层本分**:看板是数据源的镜子,不是新的分析。任何看板里的数字/结论都要能回溯到用户引用的内容;数据源里没有的不补、不猜、不编。
- **动态匹配,不查表**:用哪个模板、数据源在哪,由"提示词 + 引用内容 + 已存在模板的自描述"动态判断;skill 不预设报告类型、不内置映射表。要改呈现就改模板,不要在生成时临时自创架构导致同一模板每次长得不一样。
- **移动端体验**:手机竖屏、单栏、首屏结论、深色高对比、触摸友好、单文件自包含可离线打开。
- **诚实标注缺失**:数据缺失按模板规则显式占位,不掩盖、不补全。

## 引用文件

- `templates/*.html`——已存在的看板模板资源(每个模板自描述面向什么数据/报告,并规定架构 / 模块描述 / 缺失处理 / 样式);模板契约见 `templates/README.md`
- `../../../references/boundaries.md`——共享边界(§2 不构成投资建议、§3 分栏、§7 技术面深度与不编造数字等,渲染时原样保留)
- 数据源 = 用户在提示词中引用的报告/数据内容(通常由各复盘/分析skill产出于 `output/` 或 `data/`,具体以用户引用的文件为准)

## 与其他skill的关系

- **各复盘/分析skill**:它们产出的报告通常落在 `output/` 或 `data/`。本skill是它们的下游展示层,只读**用户引用的**那份报告文件,不调用它们的取数/分析流程,也不预设"某报告类型一定在某个路径"。用户未引用具体文件时,提示用户指出文件或粘贴正文。
- **`ashare-data-source-config`**:本skill不取实时行情,因此不读 `data-source-config.json`、不触发取数配置校验。看板里的所有数据均来自用户引用的已落盘内容。
