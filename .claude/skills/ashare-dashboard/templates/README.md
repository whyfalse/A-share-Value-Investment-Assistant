# 看板模板资源契约

本文件夹存放看板模板资源。每个模板是一个**独立的 HTML 文件**,由用户自行创建/维护。本 skill(`ashare-dashboard`)**不预设报告类型、不内置"类型→模板"映射表**,而是扫描本文件夹下实际存在的模板,结合用户提示词与引用内容**动态匹配**。因此每个模板必须**自描述**自己面向什么数据/报告,供 skill 匹配。

> 本 README 是模板契约说明,不是模板本身。模板 HTML 文件由你逐个创建,放在本文件夹即可。**所有模板统一面向移动端竖屏阅读设计**——手机单栏、首屏即结论、深色高对比、触摸友好。

## 模板自描述(必需)

每个模板 HTML 在文件首部用 HTML 注释声明元信息,供 skill 读取匹配:

```html
<!--
@dashboard-template
name: 晚间复盘看板
match: 晚间复盘,收盘复盘,盘后复盘
target: A股收盘后晚间复盘报告
data: 指数表现,领涨领跌板块,资金面,持仓四维卡片,对称情景,消息面,观察清单
-->
```

字段说明:
- `name`:模板展示名。
- `match`:匹配关键词,逗号分隔——用户提示词或引用内容命中时倾向选本模板。
- `target`:本模板面向的数据/报告类型,一句话说明。
- `data`:本模板期望渲染的数据单元,逗号分隔。

> 文件名**自由命名**(如 `evening_review.html`、`智普.html`),skill 按自描述匹配、**不按文件名查表**。无自描述时 skill 会退而读 `<title>` 与结构推断用途,但匹配不可靠,建议每个模板都写自描述。

## 每个模板 HTML 必须规定的内容

1. **看板架构**:看板分哪些区/卡片、先后顺序与层级、移动端首屏放什么。用**真实可渲染的 HTML 结构**表达(可用占位数据示意),不要只写文字说明。架构须遵循下方「内容设计指导」的信息分层原则(总览结论 → 细分数据 → 辅助信息)。
2. **模块描述**:每个模块需说明
   - 展示哪部分数据(对应数据源的哪个章节 / 表格 / 字段)
   - 用什么形式展示(数字大字 / 表格 / 横向条形 / 迷你K线 / 标签徽章 / 文字段落 等)
   - **数据缺失处理逻辑**:该模块某项数据缺失时显示什么(占位符"—" / 灰显 / "数据缺失"标注 / 整块降级)。建议在每个模块用 HTML 注释标注,例如:`<!-- 缺失时: 显示"—"并灰显该行 -->`
3. **样式约束**:配色(涨红跌绿 / 面板色 / 文字层级色)、字体、圆角、间距、移动端断点。用 `:root` CSS 变量集中定义,便于各次生成视觉统一。具体移动端渲染规格见下方「移动端设计规范」,内容排布原则见「内容设计指导」。

## 移动端设计规范

模板与最终看板都必须按**手机竖屏阅读**优化。以下为硬性规格,创建/校验模板时逐条对照:

**视口与版心**
- 必含 `<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">`。
- 主体**单栏**布局,最大宽度 `≤ 480–560px`,左右居中;大屏(≥768px)留白居中即可,**不主动展开成多栏**。
- 适配刘海与底部手势条:容器 `padding` 走 `env(safe-area-inset-*)`,关键操作不压在安全区外。

**字号与触摸**
- 正文 ≥ 15px,标签/辅助小字 ≥ 12px;核心大数字 28–40px。行高 1.5–1.7。
- 可点击元素热区 ≥ 44×44px,相邻可点元素间距 ≥ 8px。

**首屏优先**
- 首屏不放占满屏的标题 banner;**首屏即给核心结论 / 总览数字**(当日涨跌、关键判断)。
- 次要内容下沉、折叠或小字,保证打开即见要点。

**配色与对比**
- 深色背景 + 高对比文字,便于夜间阅读;涨红跌绿沿用 A 股惯例,但**颜色之外叠加箭头/图标双编码**,兼顾色弱。
- 文字层级用 CSS 变量分 3–4 级(主标题/正文/辅助/弱化),不单靠亮度差区分。

**禁横向滚动**
- 表格、图表宽度自适应屏宽;**长表格转卡片列表或横向条形**,不在移动端塞整屏表格。
- 超长数字用千分位/缩写(亿/万),不撑破版心。

**自包含与性能**
- CSS 内联、字体可引公共 CDN、图标优先内联 SVG/emoji;单文件可离线打开,不依赖本地额外资源。

**推荐骨架**(可直接作为模板起点,展示上述规格如何落地):
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
<title>晚间复盘看板</title>
<style>
  :root{
    --bg:#0f1419; --panel:#1a2029; --text:#e6e8eb; --sub:#8a949e; --weak:#5b6570;
    --up:#e53935; --down:#43a047; --accent:#3d7eff; --radius:12px; --gap:12px;
  }
  *{box-sizing:border-box; margin:0; padding:0;}
  body{background:var(--bg); color:var(--text);
       font-family:-apple-system,"PingFang SC","Microsoft YaHei",sans-serif;
       font-size:15px; line-height:1.6; max-width:520px; margin:0 auto;
       padding:calc(env(safe-area-inset-top) + 12px) 14px calc(env(safe-area-inset-bottom) + 24px);}
  .card{background:var(--panel); border-radius:var(--radius); padding:14px; margin-bottom:var(--gap);}
  .h1{font-size:18px;} .kpi{font-size:32px; font-weight:700;}
  .up{color:var(--up);} .down{color:var(--down);} .sub{color:var(--sub); font-size:13px;}
  a,button,.tap{min-height:44px; display:inline-flex; align-items:center;} /* 触摸热区 */
</style>
</head>
<body>
  <header><h1 class="h1">晚间复盘 · 06-28</h1></header>
  <!-- 首屏核心:结论先行 -->
  <section class="card">
    <div class="sub">上证指数 收盘</div>
    <div class="kpi up">3,287.42 ▲0.86%</div>
    <div class="sub">成交 6,601亿 · 较前日 +12%</div>
  </section>
  <!-- 后续板块:细分 → 辅助,逐块下沉 -->
</body>
</html>
```

## 内容设计指导

模板的内容排布应让用户**扫一眼就抓到重点**,不要把信息平铺。以下为设计原则,创建/修改模板时对照执行;skill 生成时沿用模板既有的分层与重点设置,不临时重排。

**1. 板块排版直观:信息分层 + 视觉分组**
- 按「**总览结论 → 细分数据 → 辅助信息**」三层从上到下排:先给当日整体判断与关键数字,再展开板块/个股,最后放消息面、观察清单。
- 每个板块有**明确标题 + 一句话定位**(标题下用小字说本块看什么),让用户知道每块在讲什么。
- 板块间用**卡片 / 留白 / 分隔线**划清边界,同类信息聚一块,跨类不混排;一张卡片只讲一件事。

**2. 图表文字结合:每图配结论,不让图自说自话**
- 每张图表**紧跟一行"一句话结论"**(takeaway),点出趋势/拐点/异常,避免用户自己猜图意。
- 趋势类用「**迷你 sparkline + 当日数值 + 涨跌幅**」三件套,小屏不画完整 K 线占满屏。
- 对比类用**横向条形 + 数值标注**(板块涨跌幅、资金净流入排行),避免竖向柱状图在窄屏挤成一团。
- 图表高度受控(约 60–120px),结论文字在其下——图为主、文点睛。

**3. 重点内容重点展示:大字号 + 强调色 + 好位置**
- 核心结论与关键数字:**大字号 + 强调色 + 首屏位置**,从"可选阅读"里挑出来单独成块。
- 用卡片/色块/徽章区分"必看"与"可看";纪律措辞(参考位"非下单指令"、对称情景双向、风险提示)用**独立弱化样式**(小字 + 灰/边框)与数据区分,既保留又不喧宾夺主。
- 涨跌方向**颜色 + 箭头双编码**,方向一眼可辨,色弱也能读。

**4. 信息密度控制:不堆砌,主次分明**
- 单屏 3–5 个信息单元为宜;次要信息折叠、小字或下沉,主信息放大。
- 长列表(板块/个股)取 **Top N + 折叠/查看更多**,不在首屏铺几十行。
- 表格在移动端转卡片或横向条;非要表格则只留 3–4 列,超长数值缩写。

## 约定

- 模板里的数字 / 文字为**示意占位**,skill 生成时用数据源真实数据替换;架构与样式保留。
- 模板**移动端优先**:按「移动端设计规范」逐条落地——viewport、单栏窄版心、触摸友好字号、深色高对比、首屏结论、禁横向滚动。
- 模板的内容排布遵循「内容设计指导」:板块分层直观、图表配结论、重点单独成块、密度克制。
- 模板应**单文件自包含**:CSS 内联,字体可引公共 CDN,图标优先内联 SVG / emoji。
- 模板须保留纪律措辞位:参考位"非下单指令"、对称情景双向、短期机会与中长期持仓分栏、风险提示与免责声明——这些位置在生成时由数据源内容填充,但结构要在模板里预留。
