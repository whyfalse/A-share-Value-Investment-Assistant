# 看板模板资源契约

本文件夹存放看板模板资源。每个模板是一个**独立的 HTML 文件**,由用户自行创建/维护。本 skill(`ashare-dashboard`)**不预设报告类型、不内置"类型→模板"映射表**,而是扫描本文件夹下实际存在的模板,结合用户提示词与引用内容**动态匹配**。因此每个模板必须**自描述**自己面向什么数据/报告,供 skill 匹配。

> 本 README 是模板契约说明,不是模板本身。模板 HTML 文件由你逐个创建,放在本文件夹即可。

## 模板自描述(必需)

每个模板 HTML 在文件首部用 HTML 注释声明元信息,供 skill 读取匹配:

```html
<!--
@dashboard-template
name: 晚间复盘看板
match: 晚间复盘,收盘复盘,盘后复盘
target: A股收盘后晚间复盘报告
data: 指数表现,领涨领跌板块,资金面,持仓四维卡片,对称情景,消息面,观察清单
-->
```

字段说明:
- `name`:模板展示名。
- `match`:匹配关键词,逗号分隔——用户提示词或引用内容命中时倾向选本模板。
- `target`:本模板面向的数据/报告类型,一句话说明。
- `data`:本模板期望渲染的数据单元,逗号分隔。

> 文件名**自由命名**(如 `evening_review.html`、`智普.html`),skill 按自描述匹配、**不按文件名查表**。无自描述时 skill 会退而读 `<title>` 与结构推断用途,但匹配不可靠,建议每个模板都写自描述。

## 每个模板 HTML 必须规定的内容

1. **看板架构**:看板分哪些区/卡片、先后顺序与层级、移动端首屏放什么。用**真实可渲染的 HTML 结构**表达(可用占位数据示意),不要只写文字说明。
2. **模块描述**:每个模块需说明
   - 展示哪部分数据(对应数据源的哪个章节 / 表格 / 字段)
   - 用什么形式展示(数字大字 / 表格 / 横向条形 / 迷你K线 / 标签徽章 / 文字段落 等)
   - **数据缺失处理逻辑**:该模块某项数据缺失时显示什么(占位符"—" / 灰显 / "数据缺失"标注 / 整块降级)。建议在每个模块用 HTML 注释标注,例如:`<!-- 缺失时: 显示"—"并灰显该行 -->`
3. **样式约束**:配色(涨红跌绿 / 面板色 / 文字层级色)、字体、圆角、间距、移动端断点。用 `:root` CSS 变量集中定义,便于各次生成视觉统一。

## 约定

- 模板里的数字 / 文字为**示意占位**,skill 生成时用数据源真实数据替换;架构与样式保留。
- 模板应**移动端优先**:`viewport`、单栏/窄栏、触摸友好字号、深色高对比。
- 模板应**单文件自包含**:CSS 内联,字体可引公共 CDN,图标优先内联 SVG / emoji。
- 模板须保留纪律措辞位:参考位"非下单指令"、对称情景双向、短期机会与中长期持仓分栏、风险提示与免责声明——这些位置在生成时由数据源内容填充,但结构要在模板里预留。
