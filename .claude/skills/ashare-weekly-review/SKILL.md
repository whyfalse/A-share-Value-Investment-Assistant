# A股周复盘(周日上午)

## 适用边界(Scope) —— 必须遵守

1. **这是"周度深度复盘",不是"每日复盘的重复"**:目的是看清一周的整体面貌、清算积压的未结事项,不是逐日复述`ashare-premarket-review`/`ashare-intraday-review`已经做过的内容。若某条信息当天已经处理清楚,这里只做一句话归纳,不重新展开。
2. **不读取、不依赖用户的历史交易记录**:本skill复盘的对象是持仓股、自选股、关注板块的**当前快照**,以及本周内发生的客观市场/消息事实,不需要也不应该尝试还原用户本周具体买卖了什么、何时下单、成交价格等交易流水信息。持有期窗口判断只用持仓快照里已有的`buy_date`字段做静态计算,不涉及完整交易历史。
3. **最低持有纪律仍然适用**:本周表现不佳的持仓,只有当核心逻辑(基本面/估值)被证实真正发生变化时才考虑调整;单纯的周度股价波动不构成调整理由。这条边界与另外两个日常复盘skill完全一致。
4. **不做新的个股深度技术分析**:若复盘中发现某持仓/自选股需要更深入的技术面研判,提示用户另行触发`ashare-technical-analysis`,不在本skill内展开。
5. **善用"全球市场休市"这个特性**:周日上午没有需要追的实时行情,本环节应聚焦"已发生的一周"和"已知的下周日程",不临时拉取新的盘前信息——那是`ashare-premarket-review`周一早上的职责。
6. **不设硬性时长上限,但要可持续重复**:建议自行控制在一个可重复的范围内(例如30-60分钟),宁可每周都能做完,也不要追求一次做到完美。

## 所需本地输入

| 输入 | 内容 | 来源 |
|---|---|---|
| 持仓记录(当前快照) | 用于持仓复盘,仅使用其中的成本、止盈止损线、买入日期等静态字段,不还原交易过程 | `ashare-intraday-review` 维护的 `positions` 本地文件 |
| 自选股记录 | 用于自选股复盘 | `ashare-intraday-review` 维护的 `watchlist` 本地文件 |
| 关注板块清单 | 用于板块复盘,代表用户主动跟踪的行业视角 | 本skill维护,见 `assets/sector_watchlist_template.json` |
| 短期机会记录 | 用于短期机会战绩复盘,仅判断催化剂是否兑现/证伪(市场事实),不涉及用户是否实际交易 | `ashare-intraday-review` 维护的 `short_term_watch` 本地文件 |
| 宏观环境记忆(含候选维度) | 用于宏观环境记忆维护 | `ashare-macro-context` 维护的本地记忆文档 |
| 本周每日复盘的累积记录 | 用于核对本周积压的未结事项,以及作为热点新闻周度汇总的原始素材(若每日`ashare-premarket-review`保存了"过去24小时热点新闻"部分) | 若用户保留了本周`ashare-premarket-review`/`ashare-intraday-review`的输出,据此核对/汇总;若未保留,请用户回忆补充,或基于可获得的信息重新梳理本周热点

若关注板块清单文件不存在,提示用户先用 `assets/sector_watchlist_template.json` 建立,不要凭空假设用户在关注哪些板块。

## 复盘内容

### 1. 上周大盘回顾(泛市场层面背景)
- 周度指数表现(上证、深证、创业板、北证等周涨跌幅)
- 周度资金流向:北向/南向资金本周净流向趋势
- 市场情绪指标:涨跌停家数、换手率等周度变化
- 这一块只提供大盘层面的参照系,具体板块的深入复盘见第5部分,这里不重复展开

### 2. 上周国内/全球热点消息汇总
- **时间窗口**:覆盖上一个周日(上次周复盘)至本次复盘的完整一周,与每日`ashare-premarket-review`的"过去24小时"窗口首尾相接,合起来正好覆盖整周,不留缺口
- **不是逐日新闻的简单拼接,而是周度提炼**:若某个事件本周持续演变(如某项政策从传闻到落地),应整合成一条"事件本周进展脉络",而不是按天罗列重复信息
- **分类整理**(与每日复盘保持同一套分类,便于衔接):国内政策与监管 / 行业与产业动态 / 公司层面 / 国际重大事件
- **国内外分栏**:国内热点和全球热点分别汇总,不混在一起,方便区分哪些是直接影响A股、哪些是间接外溢影响
- 范围限定与每日复盘一致:只收录与市场、宏观环境或持仓/自选股/关注板块相关行业有关的信息,不做泛新闻聚合
- 与第6部分(宏观环境记忆维护)联动:本周持续出现、具备 Tier 3 级别影响的事件,在此一并确认是否已经/需要写入宏观记忆

### 3. 持仓股复盘(核心)
- 逐一核对每个持仓本周表现 vs 大盘/相关板块,判断本周波动主要是个股 alpha 还是行业/大盘 beta
- **核心逻辑核实**:本周是否出现任何新信息(公告、行业政策、宏观背景变化)实质性改变了持有 thesis——区分"逻辑变了"和"股价波动了",只有前者才是值得关注的信号
- **持有期窗口提示**:基于`positions`快照中的`buy_date`,标注是否有持仓本周内达到/超过最低持有天数,进入"可以重新评估"的窗口——这只是提示窗口已打开,不代表必须操作,也不需要知道用户期间是否做过任何交易
- 仓位结构周度回顾:基于当前快照中的`position_pct`,是否明显偏离自己设定的上限

### 4. 自选股复盘
- 逐一核对自选股本周是否触及预设买点条件(`target_buy_condition`)
- 若本周出现新信息,判断买点条件本身是否需要更新(更新条件 ≠ 临时放宽标准去追涨)

### 5. 关注板块复盘
- 对照`assets/sector_watchlist_template.json`中的关注板块清单,逐一复盘:
  - 本周该板块整体表现,以及背后的客观背景(政策、产业事件、资金流向等,可直接引用第2部分已汇总的内容,不重复梳理)
  - 该板块的长期关注理由(`reason_for_interest`)本周是否有新信息强化或削弱
  - 交叉关联`related_holdings_or_watchlist`字段中的具体持仓/自选股,提示是否需要在第3、4部分重点留意
- 与 `ashare-macro-context` 的"M_长期产业热点"维度互相印证:关注板块清单是用户个人视角的跟踪对象,宏观记忆中的M类是更客观的产业趋势背景,两者可以互相支撑,但不要混淆成同一份文档

### 6. 短期机会战绩复盘
- 本周到期/兑现/证伪的短期机会逐一复核,更新`short_term_watch`中对应条目的`status`
- 复核重点是判断逻辑是否站得住脚(是否靠运气蒙对),不是统计盈亏金额——盈亏依赖交易记录,本skill不涉及

### 7. 宏观环境记忆维护
- 检查 `ashare-macro-context` 记忆文档中 `next_review_due` 已到期或即将到期的条目,周日上午是统一处理的合适时机
- 检查 `candidates` 候选维度暂存区是否有条目 `decision_due` 已到,该做转正/淘汰/合并的决定
- 汇总本周每日复盘以及第2部分热点新闻汇总中标记的"建议补充进宏观记忆"事项,统一核实并实际写入——周复盘有充足时间,建议直接执行`ashare-macro-context`的维护流程完成更新,而不只是标记建议

### 8. 未结事项清算
- 汇总本周 `ashare-premarket-review`/`ashare-intraday-review` 积压、始终没有结论的"今日观察清单"/"待办"项
- 逐条做出明确决定:已解决(说明结论)/ 转为长期跟踪(移入持仓/自选股/关注板块对应清单)/ 确认无效可放弃
- 不允许同一事项无限期停留在"观察中"状态而没有人去处理

### 9. 下周事件预告
- 全面列出下周已知的日程:经济数据发布、央行/重要会议、政策窗口、相关持仓的财报/公告窗口、国际重大事件(如有)等
- 只标注时间和议题,不预测结果或市场反应——这是"已知的不确定性"清单,不是预测报告
- 这份预告是下周每个交易日`ashare-premarket-review`的起点:周一早间复盘可以直接核对清单中哪一项今天需要关注,不必从零开始排查

### 10. 下周重点关注的板块或个股
- 基于本周复盘(第3-5部分)的发现,明确列出下周值得重点跟踪的具体板块或个股,并说明理由(如"该板块本周出现政策催化,下周观察后续落地情况"),不是泛泛的"继续关注市场"
- 与第9部分区分:第9部分是客观日历,这里是基于本周分析得出的个人化关注重点

## 输出格式

按以下结构输出(模板见 `assets/weekly_review_template.md`):

```
## 周复盘 [周一日期 - 周五日期]

### 大盘回顾
- 指数周涨跌幅 / 资金流向 / 情绪指标

### 上周国内/全球热点消息汇总
- 国内热点:
  - 政策与监管:[摘要]
  - 行业与产业动态:[摘要]
  - 公司层面:[摘要]
- 全球热点:
  - 国际重大事件:[摘要]

### 持仓复盘
- [标的]: 本周表现(alpha/beta判断) / 逻辑是否变化 / 持有期窗口提示 / 结论

### 自选股复盘
- [标的]: 是否触及预设买点 / 买点条件是否需要更新

### 关注板块复盘
- [板块]: 本周表现与背景 / 长期关注理由是否被强化或削弱 / 关联持仓与自选股提示

### 短期机会战绩
- [标的]: 结果(confirmed/invalidated) + 过程评价(逻辑是否站得住脚)

### 宏观环境维护
- 已完成的更新事项 / 候选维度转正或淘汰决定

### 未结事项清算
- [事项]: 决定(已解决/转长期跟踪/放弃),及理由

### 下周事件预告
- [时间] [议题],仅标注待观察,不预测结果

### 下周重点关注的板块/个股
- [标的/板块]: 关注理由(来自本周复盘的具体发现)
```

## 与其他skill的关系

- **`ashare-macro-context`**:周复盘是执行其维护流程的最佳周期性时机——每日复盘只做标记,周复盘负责把标记实际落地为记忆更新,并处理候选维度的转正/淘汰决定;关注板块复盘、热点新闻汇总也会与该skill的维度框架(尤其M类、L类)互相印证。
- **`ashare-premarket-review` / `ashare-intraday-review`**:形成双向闭环——周复盘清算这两者逐日积累但未给出结论的事项,同时把"上周热点新闻"汇总成周度脉络;反过来,周复盘产出的"下周事件预告"是下周每日早间复盘的起点,避免每天从零排查日程。
- **`ashare-technical-analysis`**:若周复盘发现某标的需要更深入的技术面研判,提示用户另行触发,不在本skill内展开。
