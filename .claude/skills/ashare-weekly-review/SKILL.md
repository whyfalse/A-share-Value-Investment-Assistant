# A股周复盘(周日上午)

## 适用边界(Scope) —— 必须遵守

本skill遵守 `../../../references/boundaries.md` 的全部共享边界（最低持有纪律、不构成投资建议、短期/中长期分栏、消息面范围限定、宏观背景引用边界、不读取交易记录等），以下仅列出本skill特有的边界：

1. **这是"周度深度复盘",不是"每日复盘的重复"**:目的是看清一周的整体面貌、清算积压的未结事项,不是逐日复述`ashare-morning-brief`/`ashare-intraday-review`已经做过的内容。若某条信息当天已经处理清楚,这里只做一句话归纳,不重新展开。
2. **不做新的个股深度技术分析**:若复盘中发现某持仓/自选股需要更深入的技术面研判,提示用户另行触发`ashare-technical-analysis`,不在本skill内展开。
3. **善用"全球市场休市"这个特性**:周日上午没有需要追的实时行情,本环节应聚焦"已发生的一周"和"已知的下周日程",不临时拉取新的盘前信息——那是`ashare-morning-brief`周一早上的职责。
4. **不设硬性时长上限,但要可持续重复**:建议自行控制在一个可重复的范围内(例如30-60分钟),宁可每周都能做完,也不要追求一次做到完美。

## 所需本地输入

| 输入 | 内容 | 来源 |
|---|---|---|
| 持仓记录(当前快照) | 用于持仓复盘,仅使用其中的成本、止盈止损线、买入日期等静态字段,不还原交易过程 | `ashare-intraday-review` 维护的 `positions` 本地文件 |
| 自选股记录 | 用于自选股复盘 | `ashare-intraday-review` 维护的 `watchlist` 本地文件 |
| 关注板块清单 | 用于板块复盘,代表用户主动跟踪的行业视角 | 本skill维护,见 `../../../templates/sector_watchlist_template.json` |

| 宏观环境记忆(含候选维度) | 用于宏观背景校准,判断本周事件是否已被既有背景覆盖 | `ashare-macro-context` 维护的本地记忆文档 |
| 本周每日复盘的累积记录 | 用于核对本周积压的未结事项,以及作为热点新闻周度汇总的原始素材(若每日`ashare-morning-brief`保存了"过去24小时热点新闻"部分) | 若用户保留了本周`ashare-morning-brief`/`ashare-intraday-review`的输出,据此核对/汇总;若未保留,请用户回忆补充,或基于可获得的信息重新梳理本周热点

若关注板块清单文件不存在,提示用户先用 `../../../templates/sector_watchlist_template.json` 建立,不要凭空假设用户在关注哪些板块。

## 复盘内容

### 1. 上周大盘回顾(泛市场层面背景)
- 周度指数表现(上证、深证、创业板、北证等周涨跌幅)
- 周度资金流向:北向/南向资金本周净流向趋势
- 市场情绪指标:涨跌停家数、换手率等周度变化
- 这一块只提供大盘层面的参照系,具体板块的深入复盘见第5部分,这里不重复展开

### 2. 上周国内/全球热点消息汇总
- **时间窗口**:覆盖上一个周日(上次周复盘)至本次复盘的完整一周,与每日`ashare-morning-brief`的"过去24小时"窗口首尾相接,合起来正好覆盖整周,不留缺口
- **不是逐日新闻的简单拼接,而是周度提炼**:若某个事件本周持续演变(如某项政策从传闻到落地),应整合成一条"事件本周进展脉络",而不是按天罗列重复信息
- **分类整理**(与每日复盘保持同一套分类,便于衔接,完整分类标准见 `../../../references/message-classification.md`):国内政策与监管 / 行业与产业动态 / 公司层面 / 国际重大事件
- **国内外分栏**:国内热点和全球热点分别汇总,不混在一起,方便区分哪些是直接影响A股、哪些是间接外溢影响
- 范围限定与每日复盘一致:只收录与市场、宏观环境或持仓/自选股/关注板块相关行业有关的信息,不做泛新闻聚合

### 3. 持仓股复盘(核心)
- 逐一核对每个持仓本周表现 vs 大盘/相关板块,判断本周波动主要是个股 alpha 还是行业/大盘 beta
- **核心逻辑核实**:本周是否出现任何新信息(公告、行业政策、宏观背景变化)实质性改变了持有 thesis——区分"逻辑变了"和"股价波动了",只有前者才是值得关注的信号
- **持有期窗口提示**:基于`positions`快照中的`buy_date`,标注是否有持仓本周内达到/超过最低持有天数,进入"可以重新评估"的窗口——这只是提示窗口已打开,不代表必须操作,也不需要知道用户期间是否做过任何交易
- 仓位结构周度回顾:基于当前快照中的`position_pct`,是否明显偏离自己设定的上限

### 4. 自选股复盘
- 逐一核对自选股本周是否触及预设买点条件(`target_buy_condition`)
- 若本周出现新信息,判断买点条件本身是否需要更新(更新条件 ≠ 临时放宽标准去追涨)

### 5. 关注板块复盘
- 对照`../../../templates/sector_watchlist_template.json`中的关注板块清单,逐一复盘:
  - 本周该板块整体表现,以及背后的客观背景(政策、产业事件、资金流向等,可直接引用第2部分已汇总的内容,不重复梳理)
  - 该板块的长期关注理由(`reason_for_interest`)本周是否有新信息强化或削弱
  - 交叉关联`related_holdings_or_watchlist`字段中的具体持仓/自选股,提示是否需要在第3、4部分重点留意
- 与 `ashare-macro-context` 的"M_长期产业热点"维度互相印证:关注板块清单是用户个人视角的跟踪对象,宏观记忆中的M类是更客观的产业趋势背景,两者可以互相支撑,但不要混淆成同一份文档

### 6. 短期机会发掘
- 从周度视角**发掘**本周新出现的、有持续性的短期机会(跨日题材演化、周度级别的产业事件、政策窗口驱动等),仅列出"本周观察到什么、为什么值得注意"
- 用催化剂框架/事件驱动框架描述,与中长期持仓的基本面/估值判断标准分栏、不混用

### 7. 未结事项清算
- 汇总本周 `ashare-morning-brief`/`ashare-intraday-review` 积压、始终没有结论的"今日观察清单"/"待办"项
- 逐条做出明确决定:已解决(说明结论)/ 转为长期跟踪(移入持仓/自选股/关注板块对应清单)/ 确认无效可放弃
- 不允许同一事项无限期停留在"观察中"状态而没有人去处理

### 8. 下周事件预告
- 全面列出下周已知的日程:经济数据发布、央行/重要会议、政策窗口、相关持仓的财报/公告窗口、国际重大事件(如有)等
- 只标注时间和议题,不预测结果或市场反应——这是"已知的不确定性"清单,不是预测报告
- 这份预告是下周每个交易日`ashare-morning-brief`的起点:周一早间复盘可以直接核对清单中哪一项今天需要关注,不必从零开始排查

### 9. 下周重点关注的板块或个股
- 基于本周复盘(第3-5部分)的发现,明确列出下周值得重点跟踪的具体板块或个股,并说明理由(如"该板块本周出现政策催化,下周观察后续落地情况"),不是泛泛的"继续关注市场"
- 与第9部分区分:第9部分是客观日历,这里是基于本周分析得出的个人化关注重点

## 输出格式

按以下结构输出(模板见 `assets/weekly_review_template.md`):

```
## 周复盘 [周一日期 - 周五日期]

### 大盘回顾
- 指数周涨跌幅 / 资金流向 / 情绪指标

### 上周国内/全球热点消息汇总
- 国内热点:
  - 政策与监管:[摘要]
  - 行业与产业动态:[摘要]
  - 公司层面:[摘要]
- 全球热点:
  - 国际重大事件:[摘要]

### 持仓复盘
- [标的]: 本周表现(alpha/beta判断) / 逻辑是否变化 / 持有期窗口提示 / 结论

### 自选股复盘
- [标的]: 是否触及预设买点 / 买点条件是否需要更新

### 关注板块复盘
- [板块]: 本周表现与背景 / 长期关注理由是否被强化或削弱 / 关联持仓与自选股提示

### 短期机会发掘
- [标的]: 本周观察到的短期催化剂/事件驱动 → 为什么值得注意

### 未结事项清算
- [事项]: 决定(已解决/转长期跟踪/放弃),及理由

### 下周事件预告
- [时间] [议题],仅标注待观察,不预测结果

### 下周重点关注的板块/个股
- [标的/板块]: 关注理由(来自本周复盘的具体发现)
```

## 输出保存

完成上述复盘后,除了在对话中展示,还必须把完整报告落盘保存到 `../../../output/ashare-weekly-review/` 文件夹下,文件名为 `weekly_review_[本周一日期].md`(用本次复盘所覆盖那一周的周一日期,格式 `YYYY-MM-DD`,如 `weekly_review_2026-06-22.md`)。若文件夹不存在则先创建。同名文件已存在时直接覆盖。

## 与其他skill的关系

- **`ashare-macro-context`**:周复盘读取宏观记忆作为背景校准;关注板块复盘、热点新闻汇总与该skill的维度框架(尤其M类、L类)互相印证。宏观记忆的完整维护(包括消费更新队列、candidates转正/淘汰、框架级调整)由 `ashare-macro-context` 独立负责,周复盘不再内嵌该流程。若发现更新队列有积压,提示用户先触发 macro-context 消费。
- **`ashare-morning-brief` / `ashare-intraday-review`**:形成双向闭环——周复盘清算这两者逐日积累但未给出结论的事项,同时把"上周热点新闻"汇总成周度脉络;反过来,周复盘产出的"下周事件预告"是下周每日早间复盘的起点,避免每天从零排查日程。短期机会方面,周复盘从周度视角独立发掘跨日级别的新机会,与每日复盘技能(盘中/晚间/早间)各自从不同的时间粒度独立发掘。
- **`ashare-technical-analysis`**:若周复盘发现某标的需要更深入的技术面研判,提示用户另行触发,不在本skill内展开。
