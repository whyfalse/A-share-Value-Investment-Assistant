# A股周复盘(周日上午)

## 适用边界(Scope)

本skill遵守 `../../../references/boundaries.md` 的全部共享边界，正文不再逐条复述。以下仅列本skill特有边界：

1. **这是"周度深度复盘",不是"每日复盘的重复"**:目的是看清一周的整体面貌、清算积压的未结事项,不是逐日复述`ashare-morning-brief`/`ashare-intraday-review`已做过的内容。当天已处理清楚的信息这里只做一句话归纳。
2. **技术面取"周度中期视角",独立跑一次引擎,口径与晚间不同**:把时间尺度从"日"拉到"周",关注中期趋势——读 `ma60/120/250` 中期均线、相对强弱 `alpha_20d/60d`、周度量价、板块周度轮动,产出"周度对称条件式情景"(配方见 `../../../references/technical-depth-guide.md` §4.4)。若某标的需要更长篇的纯技术面专题报告,提示用户另行触发`ashare-technical-analysis`。
3. **善用"全球市场休市"这个特性**:周日上午没有需要追的实时行情,聚焦"已发生的一周"和"已知的下周日程",不临时拉取新的盘前信息(那是`ashare-morning-brief`周一早上的职责)。
4. **不设硬性时长上限,但要可持续重复**:建议控制在可重复的范围内(例如30-60分钟),宁可每周做完,也不追求一次做到完美。

## 复盘内容

> **启动前(取数准备)**:按 `../../../references/data-source-priority.md` 的"取数前配置准备"流程,先读 auto-memory 的 `data-source-config.json` 并做轻量校验;配置缺失或校验失败(关键专业工具消失/依赖缺失/`next_review_due` 过期、`version<2` 旧结构)时,先接力调用 `ashare-data-source-config` 探测本地工具并重新编排配置,再开始取数。**取数过程中记录每类数据实际使用的工具(primary 还是 fallback),供末尾"数据来源"小节使用。**

### 启动前必须声明的取数计划(gate,未完成不得进入第0步)

执行 `../../../references/data-source-priority.md` 的"取数计划表 gate"章节。本skill涉及的数据桶清单(按此清单逐桶查配置 `routing`):

- `cn_equity_quote` — 周度指数行情/资金流向/情绪指标
- `hk_us_equity` — VIX等若专业工具覆盖走此桶
- `overseas_uncovered` — VIX等若Wind不覆盖走此桶(仅定性)
- `news_policy` — 上周国内/全球热点消息汇总
- `announcement_doc` — 持仓/自选股本周公告
- `cn_tech_daily` — 第0步技术引擎算指标所需日线(周度中期视角)
- `fund_index_bond` — 关注板块代表性ETF行情(若需)

输出取数计划表后才允许进入第0步。

### 0. 技术引擎周度运行(中期视角,独立于晚间)

周复盘开始前,以"周度中期视角"独立跑一次技术引擎,口径与晚间复盘不同——晚间看日线短均线(ma5/10/20)抓当日异动,周复盘把尺度拉到"周",只关心中期趋势是否健康。调用方法见 `../../../references/technical-depth-guide.md` §二、周度配方见 §4.4。

- **运行对象**:主要指数(上证/深证/创业板/沪深300,`--asset-type index`)+ 全部持仓(`--asset-type stock`/`etf`)+ 关注板块代表性标的或板块ETF。自选股可选(若本周有明显异动或临近买点再跑)。
- **只取中期字段,忽略短均线噪音**:重点读 `ma60`(季线)/`ma120`(半年线)/`ma250`(年线)的位置与排列、`price_vs_ma250`(牛熊分水岭)、相对强弱 `alpha_20d`/`alpha_60d`(vs 沪深300)、`adx`(中期趋势强度)、周度量价(`recent_series` 近5周量价配合)、`key_levels`(中期支撑/压力)。`ma5/10`、当日金叉死叉等短周期信号在周复盘里只作旁证,不作为主线。
- **降级规则**:若引擎数据不可用(akshare 未装/取数失败),如实声明"本周技术面数据缺失",改用周K线定性描述(周阳/周阴、是否站上年线等可目测的中期特征),绝不编造指标数字(见 `../../../references/technical-depth-guide.md` §七)。

### 1. 上周大盘回顾(泛市场层面背景)
- 周度指数表现(上证、深证、创业板、北证等周涨跌幅)
- **指数中期技术面**(承接第0步引擎结果):主要指数当前相对 `ma60/120/250` 的位置(站上/跌破年线即牛熊分水岭)、中期均线排列(多头/空头/纠缠)、`adx` 反映的中期趋势强度(>40 强趋势 / <20 震荡需打折),给出一句"中期处于什么状态"的客观定性,不预测下周方向
- 周度资金流向:北向/南向资金本周净流向趋势
- 市场情绪指标:涨跌停家数、换手率等周度变化
- **VIX恐慌指数周度趋势**:周初→周末的变化方向、周均值水平(恐慌>25 / 平稳15-25 / 极度贪婪<15)——VIX反映全球资金风险偏好,是判断当周A股情绪方向与全球是否同步/背离的交叉验证参考——若VIX当周显著上行而A股仍强,说明A股在走独立情绪;若VIX与A股同步走弱,说明全球风险偏好共振
- 这一块只提供大盘层面的参照系,具体板块的深入复盘见第5部分,这里不重复展开

### 2. 上周国内/全球热点消息汇总
- **取数方式**:按 `../../../references/data-source-priority.md`,消息资讯采用"专业金融资讯/公告接口 + 联网搜索"组合交叉印证,不只用单一联网搜索敷衍成新闻流水账
- **时间窗口**:覆盖上一个周日(上次周复盘)至本次复盘的完整一周,与每日`ashare-morning-brief`的"过去24小时"窗口首尾相接,合起来正好覆盖整周,不留缺口
- **不是逐日新闻的简单拼接,而是周度提炼**:若某个事件本周持续演变(如某项政策从传闻到落地),应整合成一条"事件本周进展脉络",而不是按天罗列重复信息
- **分类整理**(与每日复盘保持同一套分类,便于衔接,完整分类标准见 `../../../references/message-classification.md`):国内政策与监管 / 行业与产业动态 / 公司层面 / 国际重大事件
- **国内外分栏**:国内热点和全球热点分别汇总,不混在一起,方便区分哪些是直接影响A股、哪些是间接外溢影响
- 范围限定与每日复盘一致:只收录与市场、宏观环境或持仓/自选股/关注板块相关行业有关的信息,不做泛新闻聚合

### 3. 持仓股复盘(核心)
- 逐一核对每个持仓本周表现 vs 大盘/相关板块,判断本周波动主要是个股 alpha 还是行业/大盘 beta
  - **用相对强弱数据坐实 alpha/beta 判断**(承接第0步):`alpha_20d`/`alpha_60d` 是中长期持仓最该关心的维度——若 alpha 持续为正,说明个股跑赢沪深300、持有 thesis 仍被市场认可;若 alpha 持续转负,说明个股开始跑输大盘,即便绝对价格上涨也只是被 beta 抬着走,值得在"核心逻辑核实"里重点追问原因。这是把"涨跌感觉"换成"客观相对强弱读数"的关键一步,不靠主观印象
- **中期均线趋势体检**(承接第0步,周度视角的核心):每只持仓读 `ma60/120/250` 的位置与排列——
  - 站上年线(`price_vs_ma250` 为正)且中期均线多头排列 → 中期趋势健康,客观描述为"中期趋势仍在轨道内"
  - 跌破年线或中期均线走平/拐头 → 客观描述为"中期趋势出现走弱迹象",作为"是否需要复查基本面"的触发信号(注意:这是提示去核实逻辑,不是卖出信号)
- **核心逻辑核实**:本周是否出现任何新信息(公告、行业政策、宏观背景变化)实质性改变了持有 thesis——区分"逻辑变了"和"股价波动了",只有前者才是值得关注的信号。技术面只回答"价格走势处于什么状态",基本面逻辑是否成立仍需独立核实,两者不可互相替代
- **周度对称条件式情景**(以中期均线为锚,双向对称,不预测、不给买卖价位):对趋势处于关键位的持仓,写成"若下周周线站稳 `ma60` 一带则中期趋势修复得到确认 / 若失守 `ma120` 则需把该持仓的基本面逻辑重新核实一遍"——两个方向都要写、措辞对称,只描述"到了哪个位置该做什么核对动作",绝不写成"大概率上涨/下跌"或给出具体买卖点位(配方见 `../../../references/technical-depth-guide.md` §4.4、§6)

### 4. 自选股复盘
- 逐一核对自选股本周是否触及预设买点条件(`target_buy_condition`)
- 若本周出现新信息,判断买点条件本身是否需要更新(更新条件 ≠ 临时放宽标准去追涨)

### 5. 关注板块复盘
- 对照`../../../templates/sector_watchlist_template.json`中的关注板块清单,逐一复盘:
  - 本周该板块整体表现,以及背后的客观背景(政策、产业事件、资金流向等,可直接引用第2部分已汇总的内容,不重复梳理)
  - **板块周度轮动持续性**(承接第0步,周度视角的板块技术读数):用板块代表性标的/板块ETF的相对强弱与中期均线,判断本周板块强弱是"延续上周趋势"还是"刚刚启动/见顶回落"——
    - 看 `alpha_20d`/`alpha_60d` 是否连续多周为正(持续轮入)还是连续转负(资金持续撤离),区分"一周脉冲"和"中期趋势"
    - 看板块ETF相对 `ma60/120/250` 的位置,给出"板块中期处于强势/修复/走弱"的客观定性
    - 板块轮动判断只描述"资金过去几周往哪儿走"的客观事实,不预测下周哪个板块会涨(轮动持续性写法见 `../../../references/technical-depth-guide.md` §5.2)
  - 该板块的长期关注理由(`reason_for_interest`)本周是否有新信息强化或削弱
  - 交叉关联`related_holdings_or_watchlist`字段中的具体持仓/自选股,提示是否需要在第3、4部分重点留意
- 与 `ashare-macro-context` 的"M_长期产业热点"维度互相印证:关注板块清单是用户个人视角的跟踪对象,宏观记忆中的M类是更客观的产业趋势背景,两者可以互相支撑,但不要混淆成同一份文档

### 6. 短期机会发掘(委托给 ashare-opportunity-discovery)
- 在第5步关注板块复盘完成后,调用 `Skill("ashare-opportunity-discovery", args="scope=medium_term")` 从周度视角发掘本周新出现的、有持续性的中长期细分板块机会,同时也可以用 `scope=short_term` 补充发掘本周跨日级别的短期催化剂机会
- 本skill第1步(大盘回顾含指数中期技术面)、第2步(本周热点消息汇总)、第5步(板块周度轮动持续性判断)的产出作为机会发掘的扫描素材
- 将发掘结果填入下方输出格式的"短期机会发掘"栏
- 发掘逻辑(催化剂框架/过滤规则/输出格式)详见该skill,此处只做调用和嵌入,不重复其逻辑

### 7. 未结事项清算
- 汇总本周 `ashare-morning-brief`/`ashare-intraday-review` 积压、始终没有结论的"今日观察清单"/"待办"项
- 逐条做出明确决定:已解决(说明结论)/ 转为长期跟踪(移入持仓/自选股/关注板块对应清单)/ 确认无效可放弃
- 不允许同一事项无限期停留在"观察中"状态而没有人去处理

### 8. 下周事件预告
- 全面列出下周已知的日程:经济数据发布、央行/重要会议、政策窗口、相关持仓的财报/公告窗口、国际重大事件(如有)等
- 只标注时间和议题,不预测结果或市场反应——这是"已知的不确定性"清单,不是预测报告
- 这份预告是下周每个交易日`ashare-morning-brief`的起点:周一早间复盘可以直接核对清单中哪一项今天需要关注,不必从零开始排查

### 9. 下周重点关注的板块或个股
- 基于本周复盘(第3-5部分)的发现,明确列出下周值得重点跟踪的具体板块或个股,并说明理由(如"该板块本周出现政策催化,下周观察后续落地情况"),不是泛泛的"继续关注市场"
- 与第8部分区分:第8部分是客观日历,这里是基于本周分析得出的个人化关注重点

## 输出格式

按以下结构输出:

```
## 周复盘 [周一日期 - 周五日期]

### 大盘回顾
- 指数周涨跌幅 / 资金流向 / 情绪指标
- VIX恐慌指数周度趋势:周初→周末变化方向 + 周均值水平(恐慌>25/平稳15-25/极度贪婪<15) + 与A股当周情绪同步/背离判断
- 指数中期技术面:主要指数 vs ma60/120/250 位置、中期均线排列、ADX 趋势强度(只定性,不预测)

### 上周国内/全球热点消息汇总
- 国内热点:
  - 政策与监管:[摘要]
  - 行业与产业动态:[摘要]
  - 公司层面:[摘要]
- 全球热点:
  - 国际重大事件:[摘要]

### 持仓复盘
- [标的]: 本周表现(用 alpha_20d/60d 坐实 alpha/beta) / 中期均线趋势体检(vs ma60/120/250) / 逻辑是否变化 / 周度对称条件式情景(以中期均线为锚,双向) / 持有期窗口提示 / 结论

### 自选股复盘
- [标的]: 是否触及预设买点 / 买点条件是否需要更新

### 关注板块复盘
- [板块]: 本周表现与背景 / 板块周度轮动持续性(alpha 连续性 + ETF vs 中期均线) / 长期关注理由是否被强化或削弱 / 关联持仓与自选股提示

### 短期机会发掘
- 由 `ashare-opportunity-discovery`（scope=medium_term 为主，可补充 scope=short_term）产出
- [标的/细分方向]: 本周观察到的催化剂/事件驱动/板块扩散 → 为什么值得注意
- (若无)本周未发掘到值得注意的新机会

### 未结事项清算
- [事项]: 决定(已解决/转长期跟踪/放弃),及理由

### 下周事件预告
- [时间] [议题],仅标注待观察,不预测结果

### 下周重点关注的板块/个股
- [标的/板块]: 关注理由(来自本周复盘的具体发现)
```

| 数据块 | 来源工具(tool_id) | 实际走的层级(primary/fallback) | 备注(失败原因/降级声明) |
|--------|---------|---------|------|
| 周度行情/资金流向 | <填配置 tool_id> | primary / fallback(写明 error.code) | `cn_equity_quote` 桶 |
| VIX周度趋势 | <填配置 tool_id> | primary / fallback | `hk_us_equity` 或 `overseas_uncovered` 桶 |
| 周度消息汇总 | <填配置 tool_id> | primary / fallback | `news_policy` 桶;**此行必须含 primary 工具,仅 web_search 时须注明"未经专业资讯源交叉核对"** |
| 技术指标(引擎计算) | 本地脚本 `compute_indicators.py` + `score.py` | primary | 基于 `cn_tech_daily` 桶日线数据 |

核对规则:每行的"来源工具"与"层级"必须与启动前取数计划表一致;若实际走了 fallback,备注列必须写明 primary 失败的 error.code。`news_policy` 一行若不含配置 primary 工具,视为未完成交叉印证,须在备注列显式声明可靠性局限。

## 输出保存

完成上述复盘后,除了在对话中展示,还必须把完整报告落盘保存到 `../../../output/ashare-weekly-review/` 文件夹下,文件名为 `weekly_review_[本周一日期].md`(用本次复盘所覆盖那一周的周一日期,格式 `YYYY-MM-DD`,如 `weekly_review_2026-06-22.md`)。若文件夹不存在则先创建。同名文件已存在时直接覆盖。
