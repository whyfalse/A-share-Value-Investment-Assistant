# 周复盘输出模板

复盘时按此结构输出。比每日复盘可以展开得更详细,但仍避免无止境堆砌——目标是做完一次完整的复盘,不是写一篇研究报告。本模板只使用持仓/自选股/关注板块的当前快照和本周客观信息,不涉及用户的交易记录。

```
## 周复盘 [周一日期 - 周五日期]

### 大盘回顾
- 指数周涨跌幅:上证XX 深证XX 创业板XX
- 资金流向:北向/南向资金本周净流向趋势
- 情绪指标:涨跌停家数、换手率等周度变化

### 上周国内/全球热点消息汇总
- 国内热点:
  - 政策与监管:[摘要,客观陈述]
  - 行业与产业动态:[摘要]
  - 公司层面:[摘要,标注与持仓/自选股/关注板块的关联]
- 全球热点:
  - 国际重大事件:[摘要]
- (若某事件本周持续演变,整合成一条进展脉络,不按天罗列)

### 持仓复盘
- [标的]: 本周表现XX(alpha/beta判断) / 核心逻辑:未变化/已变化(说明) / 持有期窗口:未到/已到 / 结论:持有/需关注
- (逐个持仓列出)

### 自选股复盘
- [标的]: 是否触及预设买点(是/否,说明) / 买点条件是否需要更新

### 关注板块复盘
- [板块]: 本周表现与客观背景(可引用上方热点汇总) / 长期关注理由是否被本周信息强化或削弱 / 关联的持仓与自选股(如有,提示需重点留意)
- (逐个关注板块列出)

### 短期机会战绩
- [标的]: 结果(confirmed/invalidated) + 过程评价(逻辑是否站得住脚,不涉及盈亏金额)
- 本期短期机会回顾完成情况:[X个已结论/Y个仍待观察]

### 宏观环境维护
- 本次已更新的维度:[列出]
- 候选维度处理:[转正/淘汰/合并的决定及理由]

### 未结事项清算
- [事项]: 决定(已解决说明结论 / 转长期跟踪移入对应清单 / 放弃说明理由)

### 下周事件预告
- [时间] [议题]——仅标注待观察,不预测结果或市场反应
- (逐项列出,作为下周每日早间复盘的起点)

### 下周重点关注的板块/个股
- [标的或板块]: 关注理由——理由必须来自本周复盘(持仓/自选股/关注板块部分)的具体发现,不是泛泛的"继续关注"
```

## 提醒事项

- 持仓/自选股/关注板块的复盘均基于当前快照状态,不需要、也不应该尝试推断用户本周具体的买卖操作或成交价格。
- 热点新闻汇总覆盖完整一周(上次周复盘至本次复盘),与每日`ashare-premarket-review`的24小时窗口首尾相接,不留缺口;但不是日新闻的简单拼接,要提炼出周度脉络。
- 未结事项清算时,凡是决定"转长期跟踪"的事项,记得同步更新到对应的本地文件(`positions`/`watchlist`/`sectors`),不要只在本次复盘文本里提一下就不再处理。
- "下周事件预告"是客观日历,不掺入个人判断;"下周重点关注的板块/个股"才是个人化的结论,两者不要混在一起写。
