# 周复盘输出模板

复盘时按此结构输出。比每日复盘可以展开得更详细,但仍避免无止境堆砌——目标是做完一次完整的复盘,不是写一篇研究报告。本模板只使用持仓/自选股/关注板块的当前快照和本周客观信息,不涉及用户的交易记录。技术面取"周度中期视角"(ma60/120/250、alpha、周度量价),只做客观状态描述+纪律线校准,不预测涨跌、不给买卖价位(边界见 `../../../references/boundaries.md` §7,写法见 `../../../references/technical-depth-guide.md` §4.4、§6)。


## 提醒事项

- 持仓/自选股/关注板块的复盘均基于当前快照状态,不需要、也不应该尝试推断用户本周具体的买卖操作或成交价格。
- 热点新闻汇总覆盖完整一周(上次周复盘至本次复盘),与每日`ashare-morning-brief`的24小时窗口首尾相接,不留缺口;但不是日新闻的简单拼接,要提炼出周度脉络。
- 未结事项清算时,凡是决定"转长期跟踪"的事项,记得同步更新到对应的本地文件(`positions`/`watchlist`/`sectors`),不要只在本次复盘文本里提一下就不再处理。
- "下周事件预告"是客观日历,不掺入个人判断;"下周重点关注的板块/个股"才是个人化的结论,两者不要混在一起写。

### 技术面三条铁律(与每日复盘一致,周度同样适用)

- **只做客观状态描述,不预测涨跌**:中期均线位置、alpha 读数、ADX 强度都是"现在处于什么状态"的事实陈述,不能写成"下周大概率上涨/下跌";ADX<20 时趋势信号本身要打折说明。
- **条件式情景必须双向对称**:写"若站稳 MA60 则…/若失守 MA120 则…"时两个方向都要给,且都落到"该做什么核对动作"(复查基本面/确认逻辑),不落到具体买卖点位,更不暗示哪个方向概率更大。
- **给区间不给点位、技术不破纪律**:支撑/压力一律用"一带""附近"的参考区间,不报精确买卖价;中期均线走弱只是"复查基本面"的触发器,不能据此对未满最低持有5日的持仓建议卖出,技术信号永远不凌驾于最低持有纪律和基本面逻辑之上。
- **数据缺失时如实声明**:引擎数据取不到就写"本周技术面数据缺失",改用周K线可目测的中期特征定性,绝不编造 MA/alpha/ADX 等指标数字。
