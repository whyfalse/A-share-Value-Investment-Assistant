# A股机会发掘

## 适用边界(Scope)

本skill遵守 `../../../references/boundaries.md` 的全部共享边界，正文不再逐条复述。以下仅列本skill特有边界：

1. **这是"发掘式输出"，不是"操作建议"**：只列出"观察到什么、为什么值得注意"，不给"应该买入/卖出"的结论（遵守 §2 不构成投资建议）。
2. **催化剂框架与基本面框架分栏**：本skill用催化剂/事件驱动框架描述机会，与中长期持仓的基本面/估值判断标准分栏、不混用（§3）。
3. **只做发现，不存储**：发掘结果仅在本次输出中呈现，不沉淀为长期清单、不改写持仓/自选股等状态文件。
4. **不突破核心持股纪律（§1）**：发掘到的短期机会不能单独成为买卖长期持仓的理由，只能"标记观察"。
5. **数据来源遵循统一优先级**：取数按 `../../../references/data-source-priority.md` 的统一优先级；消息面多源交叉印证。

## 参数说明

通过 `scope` 参数指定本次发掘聚焦的时间框架：

| scope | 时间跨度 | 发掘焦点 | 典型调用方 | 数据窗口 |
|-------|---------|---------|-----------|---------|
| `intraday` | 盘中（分钟~小时） | 量比异动、板块异动、盘中突发消息、尾盘资金抢筹/出逃 | `ashare-intraday-review` | 当日盘中实时数据 |
| `short_term` | 短期（小时~数天） | 情绪热点、隔夜消息催化、政策催化、事件驱动 | `ashare-morning-brief` / `ashare-evening-review` | 隔夜/当日全天 |
| `medium_term` | 中长期（数天~数周） | 主行情中新细分板块、跨日题材演化、周度产业事件、政策窗口 | `ashare-weekly-review` | 周度累积数据 |
| `all` | 全扫描 | 同时覆盖以上三个时间维度 | 独立使用 | 组合窗口 |

若未指定 `scope` 参数，默认行为：若在盘中时段（9:30-15:00）则默认为 `intraday`；若在收盘后~次日开盘前则默认为 `short_term`；周末则默认为 `medium_term`。

## 所需本地输入

| 输入 | 内容 | 用途 |
|------|------|------|
| 持仓记录 | 代码、名称、核心逻辑摘要 | 交叉标注机会与已有持仓的关联 |
| 自选股记录 | 代码、名称、关注理由 | 交叉标注机会是否与自选股池重叠 |
| 宏观环境记忆 | 由 `ashare-macro-context` 维护 | 判断机会催化剂是否已被既有宏观背景覆盖 |

| 行情/消息数据（按scope） | 内容 | 取数方式 |
|--------------------------|------|---------|
| 日内盘中数据（intraday） | 实时资金流向、涨跌榜、量比异动、盘中消息 | 按 `../../../references/data-source-priority.md`，优先专业金融数据工具实时获取 |
| 日间数据（short_term） | 当日/隔夜消息汇总、板块资金流、政策公告、全球市场表现 | 专业金融资讯接口 + 联网搜索交叉印证 |
| 周度累积数据（medium_term） | 周度板块轮动、跨日题材脉络、周度量价、产业事件日历 | 专业金融数据工具取周度统计 + 联网搜索补充产业动态 |

若持仓/自选股或宏观记忆文件不存在，提示用户先用对应skill建立（可直接用对应模板新建），不要凭空假设。若某类行情数据获取失败，按 `data-source-priority.md` 的降级规则处理，在输出中标注数据缺失范围。

## 三个时间框架的发掘框架

### 一、日内机会发掘（scope=intraday）

**发掘目标**：当日盘中新出现、可能在剩余交易时间或次日发酵的短期催化剂。

**信号类型**：
- **量价异动**：单只或板块突然放量（量比>3）+ 方向明确（急拉/急跌），且不能仅用大盘 beta 解释
- **板块共振**：同一概念/行业内多只标的同步异动，暗示有题材消息或资金共识正在形成
- **盘中消息驱动**：交易时段内突发公告、政策吹风、产业链新闻等，盘面已有初步反应
- **尾盘信号**：14:30后资金明显抢筹或出逃的标的/板块，可能预示次日开盘方向

**发掘方法**：
1. 结合调用方传来的"大盘/资金流向/板块温度计"结果（`ashare-intraday-review` 第4步产出），识别当日市场风格和热点方向
2. 扫描涨跌幅榜前列、量比异动榜，筛选与用户持仓/自选股同行业或同题材的标的
3. 交叉核对盘中突发消息与盘面反应是否同步
4. 对每个信号标注"距收盘还有多少时间"及"是否可能在尾盘或次日发酵"

**过滤规则**：
- 排除纯大盘 beta 驱动的普涨普跌（用板块涨跌幅 vs 大盘涨跌幅区分）
- 排除日常波动范围内的微小异动（无催化剂的 ±2% 以内波动不列为机会）
- 短期机会与中长期持仓分栏标注，不混用判断标准

### 二、短期机会发掘（scope=short_term）

**发掘目标**：隔夜或全天出现的、由情绪/消息/政策驱动的短期催化剂（通常数小时到数天的发酵期）。

**信号类型**：
- **政策催化**：国务院/部委/地方政府新出台的产业政策、监管变化、税收优惠等，已引起市场关注
- **消息面催化**：产业链重大新闻（涨价/缺货/技术突破/海外对标大涨）、公司公告（订单/合作/回购/业绩预告）
- **情绪热点**：社交媒体/财经媒体集中讨论的主题，有资金跟进迹象
- **跨市场传导**：隔夜美股/欧股某板块大涨大跌 → A股对应板块有联动预期
- **事件驱动**：即将发生的已知事件（展会/发布会/政策生效日/财报披露窗口）引发提前布局

**发掘方法**：
1. 梳理调用方传来的当日/隔夜消息面汇总（按 `message-classification.md` 四类分组），逐条判断是否有可交易的催化剂属性
2. 对每条潜在催化剂，检查对应板块/个股的盘面反应（是否有资金跟进，还是纯消息无反应）
3. 区分"已被定价"和"仍在发酵中"的催化剂——已经反应充分的（当日已涨停/已大幅上涨且缩量）降级标注
4. 交叉核对持仓/自选股：标注哪些催化剂与已有标的直接相关

**过滤规则**：
- 排除"纯消息、无资金跟进"的空洞催化剂（消息出了但盘面毫无反应）
- 排除已充分定价的催化剂（如当日已封板且无开板迹象的，不列为"新机会"）
- 排除与宏观背景无关的碎片化行业小新闻
- 不把单只个股的日常公告（如例行董事会决议）当作催化剂

### 三、中长期机会发掘（scope=medium_term）

**发掘目标**：从周度/中期视角发掘主行情中正在形成的新细分板块或方向，具有跨日~数周的持续性潜力。

**信号类型**：
- **主行情新细分方向**：当前市场主线（如AI/新能源/消费复苏等）内部，资金正在向新的细分环节扩散（如从算力扩散到散热/光模块/存储）
- **跨日题材演化**：某个题材从"一日游"演化为"持续发酵"——连续多日出现在涨幅榜、新闻联播持续提及、研报密集覆盖
- **板块轮动新方向**：资金从过热的主线流出后，正在流入的新承接板块（有量价配合，非一日游）
- **周度产业事件/政策窗口**：下周已知的产业事件（展会/发布会/标准制定会议）可能催化的细分方向
- **中期均线确认的板块趋势**：板块代表性标的/ETF站上 MA60/120 且 alpha 连续多周为正

**发掘方法**：
1. 承接调用方传来的周度板块轮动持续性判断（`ashare-weekly-review` 第5步产出），在已有轮动结论上进一步向下挖掘细分方向
2. 看板块内部结构：领涨板块内，哪些细分环节的标的最近一周才开始放量走强（而非一周前就已经涨了很多的）→ 这是"新扩散"信号
3. 交叉核对本周热点消息汇总，找出"本周多次出现、且有资金持续跟进"的跨日题材
4. 结合宏观环境记忆的 M_长期产业热点维度，判断新方向是否有产业趋势支撑（有产业趋势背书的细分方向持续性好于纯题材炒作）

**过滤规则**：
- 排除"一周脉冲"：仅在周初涨了一天、后面四天缩量回落的，不列为有持续性的中长期机会
- 排除"已经充分定价的主线龙头"——寻找的是主行情中新扩散的细分环节，而非已经涨了很久的主线龙头本身
- 中长期机会的结论仍然是催化剂/事件驱动框架，不混入基本面估值判断

## 执行流程

> **启动前（取数准备）**：按 `../../../references/data-source-priority.md` 的"取数前配置准备"流程，先读 auto-memory 的 `data-source-config.json` 并做轻量校验；配置缺失或校验失败时，先接力调用 `ashare-data-source-config` 探测本地工具并重新编排配置，再开始取数。

1. **解析 scope 参数**：确定本次聚焦的时间框架（intraday / short_term / medium_term / all）
2. **加载本地上下文**：读取 `data/positions.json`、`data/watchlist.json`、宏观环境记忆文档，用于后续交叉标注
3. **收集行情/消息数据**：按 scope 对应的数据需求取数（遵循 `data-source-priority.md`）
4. **按框架扫描信号**：用上方对应的发掘框架逐项扫描，列出候选信号
5. **过滤与分级**：按各框架的过滤规则筛除无效信号，对保留的标注催化剂强度（强/中等/待观察）
6. **交叉关联标注**：标注每个机会与持仓/自选股的关联（直接相关 / 同行业 / 无直接关联但值得注意）
7. **产出结构化输出**：按下方输出格式写入，无机会时明确声明"本次未发掘到值得注意的机会"并简述原因（如"日内无显著催化剂事件"、"板块以普涨为主无个股alpha"）

## 输出格式

```
### 机会发掘 [scope标签]

#### 日内机会（如有）
- [标的代码 名称]: [催化剂类型] 观察到的具体异动或催化剂 → 为什么值得注意
  - 催化剂来源: [政策/消息/资金/板块联动/事件驱动]
  - 时间窗口: [盘中出现/尾盘异动/预计次日发酵]
  - 与持仓关联: [直接相关/同行业/自选股/无直接关联]
  - 注意：此为短期催化剂观察，与中长期持仓的基本面判断标准不同

#### 短期机会（如有）
- [标的代码 名称]: [催化剂类型] 观察到的具体催化剂 → 为什么值得注意
  - 催化剂来源: [政策催化/消息催化/情绪热点/跨市场传导/事件驱动]
  - 发酵程度: [仍在发酵/部分定价/接近充分定价]
  - 与持仓关联: [同上]

#### 中长期机会（如有）
- [细分方向/板块]: [驱动逻辑] 主行情中新出现的细分机会 → 为什么值得注意
  - 所属主线: [当前市场主线的哪个环节]
  - 持续性信号: [跨日量价/周度alpha/产业事件支撑]
  - 与关注板块关联: [是否在用户关注板块清单内]

（无机会时）本次未发掘到值得注意的[时间框架]短期机会：[简述原因]
```

## 输出保存

完成机会发掘后，除了在对话中展示（供调用方嵌入复盘报告），还必须把独立报告落盘保存到 `../../../output/ashare-opportunity-discovery/` 文件夹下，文件名为 `opportunity_discovery_[scope]_[日期].md`（日期用当日，格式 `YYYY-MM-DD`，如 `opportunity_discovery_intraday_2026-06-25.md`）。若文件夹不存在则先创建。同名文件已存在时直接覆盖（同一天同一scope重复发掘以最新一次为准）。

## 与其他skill的关系

- **`ashare-intraday-review`**：盘中复盘调用本skill（scope=intraday）发掘日内盘中机会，并将结果嵌入其输出的"短期机会发掘"栏。本skill接收其传来的大盘/资金流向/板块温度计结果作为扫描参照系。
- **`ashare-evening-review`**：晚间复盘调用本skill（scope=short_term）发掘全天观察到的短期催化剂机会，并将结果嵌入其输出的"短期机会发掘"栏。本skill接收其传来的全天A股复盘、板块轮动持续性判断、白天消息面汇总作为扫描素材。
- **`ashare-morning-brief`**：早间推送调用本skill（scope=short_term）发掘隔夜催化剂机会，并将结果嵌入其输出的"短期机会发掘"栏。本skill接收其传来的隔夜消息汇总和全球市场表现作为扫描素材。
- **`ashare-weekly-review`**：周复盘调用本skill（scope=medium_term）从周度视角发掘中长期细分板块机会，并将结果嵌入其输出的"短期机会发掘"栏。本skill接收其传来的周度板块轮动持续性判断、本周热点消息汇总、指数中期技术面作为扫描素材。
- **`ashare-macro-context`**：本skill读取宏观环境记忆作为催化剂背景校准（判断催化剂是否有宏观逻辑支撑），但不修改宏观记忆文件。若发掘过程中发现新的宏观事件线索，可提示用户触发 `ashare-macro-context` 更新。
- **`ashare-technical-analysis`**：本skill不直接调用技术引擎。技术面的支撑来自调用方已产出的技术读数（如晚间复盘的均线/ADX、周复盘的alpha/中期均线），本skill仅将这些技术读数作为催化剂判断的辅助参照。

## 独立使用

用户也可以不经过复盘技能，直接触发本skill做独立机会发掘：
- "发掘一下最近的[短期/中长期]机会"
- "看看有什么新机会"
- "扫描一下[日内/隔夜]的催化剂"

独立使用时，本skill自行收集所需行情/消息数据（不依赖调用方传入的汇总结果），按相同的发掘框架和过滤规则执行全流程扫描。
