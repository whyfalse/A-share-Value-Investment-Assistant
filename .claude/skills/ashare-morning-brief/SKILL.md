# A股早间消息推送(8:00-8:20窗口,集合竞价前)

## 适用边界(Scope)

本skill遵守 `../../../references/boundaries.md` 的全部共享边界，正文不再逐条复述。以下仅列本skill特有边界：

1. **这是"隔夜信息推送",不是"开盘前操作建议"**:目标是让用户开盘前掌握"隔夜欧美及全球市场怎么走、隔夜有什么相关消息、今天有什么已知日程",并据此建立今日观察清单,不给"今天该买该卖"的结论。
2. **盘前不跑技术引擎**:盘前数据少、A股尚未开盘。技术深度来自两处轻量动作:隔夜外盘的**定性**描述(涨/跌/平、是否破关键位,不硬编点位)+ 调出昨晚晚间复盘的**技术参考位**作为"今日盘中观察基准"。承接关系与写法见 `../../../references/technical-depth-guide.md` §4.2。
3. **隔夜消息窗口**:覆盖"昨晚晚间复盘之后 → 今早"(欧美交易时段这一夜),与前一晚 `ashare-evening-review` 的白天窗口首尾相接、合成完整24小时不留缝。
4. **完全不涉及亚太市场**:亚太(日韩港)由前一晚的 `ashare-evening-review` 以完整收盘数据覆盖;本skill聚焦隔夜欧美及全球其他市场。如需参照,只用一句"完整亚太数据见昨晚晚间复盘"指引,不展开。
5. **Tier 3 事件走队列**:发现 Tier 3 级隔夜事件时,产出更新建议追加至 `../../../data/macro_updates_queue.json`(协议见 `../../../references/tier3-event-handling.md`),不直接写 `macro_context.json`。

## 所需本地输入

| 输入 | 内容 | 来源 |
|---|---|---|
| 持仓/自选股记录 | 用于判断隔夜消息是否与已有标的相关 | `data/positions.json` / `data/watchlist.json` |
| 宏观环境记忆 | 用于判断隔夜事件是否已被既有背景覆盖,或属于全新信息 | `ashare-macro-context` 维护的本地记忆文档 |
| 昨晚晚间复盘的隔夜需观察事项 | 用于核对这些事项的隔夜进展,避免遗漏 | `output/ashare-evening-review/evening_review_[昨日].md` |
| 昨晚技术参考位 | 昨晚晚间复盘移交的各持仓支撑/压力/纪律线,作为今日盘中观察基准 | `../../../output/ashare-evening-review/evening_review_[昨日].md` |
| 隔夜美股/欧洲/全球行情 | 用于第1、2块内容 | 按 `../../../references/data-source-priority.md` 选择工具：优先专业金融数据工具取精确数值，通用搜索仅兜底取定性 |
| 隔夜消息/今日已知日程 | 用于第3、4块内容 | 按 `../../../references/data-source-priority.md` 选择工具：专业金融资讯/公告接口 + 联网搜索组合交叉印证，不只用单一搜索 |

若持仓/自选股或宏观记忆文件不存在,提示用户先用对应skill建立,不要凭空假设内容。若昨晚晚间复盘留下的隔夜需观察事项本次仍无法给出结论,应继续保留进"今日观察清单",而不是默默丢弃。

> 昨晚晚间复盘报告若不存在(如周一早晨、节后首日、昨晚未复盘),则跳过"今日技术参考位"的承接部分,如实写"无昨晚参考位",仅做隔夜外盘定性,不回填、不编造历史测算位(降级规则见 `../../../references/technical-depth-guide.md` §七)。

## 四块推送内容

> **启动前(取数准备)**:按 `../../../references/data-source-priority.md` 的"取数前配置准备"流程,先读 auto-memory 的 `data-source-config.json` 并做轻量校验;配置缺失或校验失败(关键专业工具消失/依赖缺失/`next_review_due` 过期)时,先接力调用 `ashare-data-source-config` 探测本地工具并重新编排配置,再开始取数。隔夜美股/欧股按配置走对应桶——美股美指若专业工具覆盖走专业桶,日股/欧股等不覆盖市场走 `overseas_uncovered` 桶(仅采信定性信息)。

### 1. 隔夜美股
- 道指、纳指、标普三大指数收盘表现
- 中国概念股/ADR整体表现(对A股开盘情绪通常有参考价值,仅作参考,不做预测)
- 美债收益率、美元指数、原油等大宗商品隔夜变化
- 是否有重要美股公司财报/事件,可能关联国内某行业板块(只标注关联可能性,不展开分析)

### 2. 欧洲及全球其他主要市场
- 欧洲市场隔夜收盘表现(此时已完整收盘,数据完整)
- 隔夜是否有全球性宏观事件(央行表态、地缘政治事件等)
- **不含亚太**:亚太市场(日韩港)由昨晚 `ashare-evening-review` 覆盖,本块不重复;如需参照,只一句话指引"完整亚太数据见昨晚晚间复盘"

### 3. 隔夜消息汇总
- **时间窗口**:覆盖"昨晚晚间复盘之后 → 今早"的隔夜时段,承接前一晚晚间复盘的白天窗口,合起来才是完整24小时,不留缝
- **范围限定**:只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息,客观陈述,不做解读延伸,不是隔夜所有新闻的流水账
- **分类整理**(便于快速扫读,也便于与 `ashare-macro-context` 维度框架对应):按 `../../../references/message-classification.md` 的统一四类分组摘要(严格按该文件规定的顺序:国内政策与监管 → 行业与产业动态 → 公司层面 → 国际重大事件),而不是按时间顺序罗列
- **与持仓/自选股的关联标注**:整理时标注哪些消息与已有持仓/自选股直接相关,作为"今日观察清单"的来源依据之一
- **核对昨晚隔夜需观察事项**:逐条核对前一晚 `ashare-evening-review` 留下的"隔夜需观察事项"是否有隔夜进展或结论,有结论的在此说明,没有的并入今日观察清单继续观察
- 判断是否有 Tier 3 级别的新宏观事件,产出更新建议追加至 `data/macro_updates_queue.json`（队列协议详见 `../../../references/tier3-event-handling.md`）,并注明建议的类别代码(如 I_国际形势、K_全球主要央行政策等)

### 4. 今日技术参考位(承接昨晚晚间复盘,盘前不现算)
- **隔夜外盘定性**:第1、2块已描述的美股三大指数、欧洲、A50期货隔夜方向,定性收口为"隔夜外盘整体偏强/偏弱/分化、是否破关键位"——只定性,不硬编点位、不预测A股开盘方向
- **持仓今日观察基准**:从昨晚晚间复盘报告里调出每只持仓的支撑/压力/纪律线,作为今日盘中观察基准,写成"XX 昨收A,昨晚测算支撑B、压力C、止损D(距昨收E%)——今日若回踩B一带可观察是否站稳"(B/C/D均为参考位,非下单指令)
- 这一步只是把昨晚已算好的位"摆出来当今日尺子",**不现算新指标、不预测涨跌**;盘中由 `ashare-intraday-review` 的"技术位互动温度计"拿实时价来对照
- (无昨晚参考位时)如实写"无昨晚参考位",仅保留隔夜外盘定性

### 5. 今日已知日程 + 今日观察清单
- **今日已知日程**:经济数据发布、公司公告窗口、政策发布会、重要会议等——这属于"已知的不确定性",只标注时间和议题,不预测结果
- **今日观察清单**(衔接14:30盘中复盘):把今日需要关注的标的/主题列出,标注关注原因(来自隔夜消息、昨晚遗留、今日日程或技术参考位),仅作观察标记、非操作建议,交棒给当日 `ashare-intraday-review` 在盘中回顾并给出结论
- 中长期持仓如遇隔夜重大消息,明确写"已标记观察,按既有纪律线处理,不建议开盘前临时操作",而不是给出操作倾向

## 输出格式

按以下结构输出,保持简洁,每条一两句话:

```
## 早间消息推送 [日期 8:00-8:20]

### 隔夜美股
- 三大指数表现 / 中国概念股表现 / 美债美元商品
- 重要美股财报/事件关联国内板块(如有,仅标注关联可能)

### 欧洲及全球其他市场
- 欧洲市场隔夜收盘表现(完整数据)
- 隔夜全球宏观事件(如有)
- (不含亚太——见昨晚晚间复盘)

### 隔夜消息汇总(昨晚晚间复盘→今早)
- 国内政策与监管:[隔夜如有]
- 行业与产业动态:[摘要]
- 公司层面:[摘要,标注与持仓/自选股的关联]
- 国际重大事件:[摘要]

### 今日技术参考位(承接昨晚,盘前不现算)
- 隔夜外盘定性:美股/欧洲/A50期货整体偏强/偏弱/分化,是否破关键位(只定性,不硬编点位、不预测A股开盘)
- 持仓今日观察基准:[标的] 昨收A,昨晚测算支撑B/压力C/止损D(距昨收E%)→ 今日若回踩B一带可观察是否站稳(均为参考位,非下单指令)
- (无昨晚参考位时)无昨晚参考位,仅做隔夜外盘定性

### 短期机会发掘
- [标的]: 隔夜观察到的短期催化剂/事件驱动 → 为什么值得注意
- (若无)隔夜未发掘到值得注意的短期机会

### 昨晚隔夜观察事项核对
- [事项]: 隔夜进展/结论(已解释/并入今日观察清单)

### 今日已知日程
- [时间] [议题],仅标注待观察,不预测结果

### 已完成宏观更新队列
- (如有Tier3新事件)已产出更新建议至队列:[类别代码 + 维度名称 + 更新内容简述]
- (若无)今日无需产出宏观更新建议的新事件

### 今日观察清单(衔接14:30盘中复盘)
- [标的/主题]: 关注原因(隔夜消息/昨晚遗留/今日日程/技术参考位),仅作观察标记,非操作建议
```

## 输出保存

完成上述推送后,除了在对话中展示,还必须把完整报告落盘保存到 `../../../output/ashare-morning-brief/` 文件夹下,文件名为 `morning_brief_[日期].md`(日期用当日,格式 `YYYY-MM-DD`,如 `morning_brief_2026-06-25.md`)。若文件夹不存在则先创建。同名文件已存在时直接覆盖(同一天重复推送以最新一次为准)。

## 与其他skill的关系

- **`ashare-evening-review`**:本skill承接其前一晚产出的"隔夜需观察事项"并核对隔夜进展,同时调出其移交的**技术参考位**(各持仓支撑/压力/纪律线)作为今日盘前观察基准;隔夜消息窗口与晚间复盘的白天窗口首尾相接,合成完整24小时闭环。短期机会方面,晚间复盘和早间推送各自从不同的时间窗口独立发掘新机会。
- **`ashare-intraday-review`**:本skill产出的"今日观察清单"与"今日技术参考位"交棒给当日 14:30-14:45 盘中复盘——盘中的"技术位互动温度计"会拿实时价对照本skill摆出的昨晚参考位,并在"宏观背景校准"和"待办"环节回顾观察清单并给出结论(已解释/仍观察/转入宏观记忆),不应让同一事项在多个skill之间无限循环却始终没有结论。三个skill共享同一套最低持有纪律,任何一方都不能单独据此突破。短期机会方面,早间推送和盘中复盘各自从不同的时间窗口(隔夜 vs 盘中)独立发掘新机会。
- **`ashare-macro-context`**:本skill对 Tier 3 级新事件产出更新建议追加至队列,由 macro-context 统一消费处理。正式条目的新建、candidates 转正、框架级维护仍由该skill负责。
- **`ashare-technical-analysis`**:本skill盘前不调用该引擎(A股未开盘、盘前数据少),技术参考位完全承接自昨晚晚间复盘已算好的结果。若早间发现某持仓/自选股因隔夜消息需要更深入的技术面研判,提示用户盘后另行触发该skill,不在20分钟窗口内展开。
