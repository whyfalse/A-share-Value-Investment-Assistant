# A股早间消息推送(8:00-8:20窗口,集合竞价前)

## 适用边界(Scope) —— 必须遵守

1. **这是"隔夜信息推送",不是"开盘前操作建议"**:目标是让用户在开盘前掌握"隔夜欧美及全球市场怎么走、隔夜有什么相关消息、今天有什么已知日程",并据此建立今日观察清单——不是给出"今天该买该卖"的结论。
2. **最低持有纪律是硬边界**:见 `../../../references/holding-discipline.md`。即使隔夜出现重大消息,中长期持仓(最低持有5个交易日)的处理方式仍是"标记观察、等待开盘后按既有纪律线核对",不因为一条隔夜新闻就建议提前操作。这一边界与 `ashare-evening-review`、`ashare-intraday-review` 一致,三个skill共享同一套持仓纪律,任何一方都不能单独据此突破。
3. **短期机会只做发掘式输出**:早间推送从隔夜催化剂中**发掘**新出现的、有明确事件驱动逻辑的短期机会,仅列出"隔夜观察到什么、为什么值得注意"。与中长期持仓分栏呈现,不混用判断标准。
4. **只汇总事实,不预测走势**:隔夜行情、今日日程都是已发生或已确定的事实;"今日已知日程"中尚未发生的项(如某经济数据将公布)只标注为"待观察的日程项",不预测数据结果或对市场的影响方向。
5. **隔夜消息有明确的时间窗口和范围限定**:隔夜消息汇总覆盖"昨晚晚间复盘之后 → 今早"的时段(欧美交易时段这一夜),与前一晚 `ashare-evening-review` 的"白天"窗口首尾相接、合起来构成完整24小时、不留缝。范围上只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息,做重点摘要,不是隔夜所有新闻的流水账。
6. **完全不涉及亚太市场**:亚太(日韩港)处于亚洲白天交易时段,由前一晚的 `ashare-evening-review` 以完整收盘数据覆盖;本skill聚焦隔夜欧美及全球其他市场。若隔夜美股/欧洲数据需要参照亚太,只用一句"完整亚太数据见昨晚晚间复盘"指引,不展开。
7. **不替代 `ashare-macro-context` 的长期记忆维护职责**:早间推送是"每日一次性简报",不直接改写宏观长期记忆文档。若发现的隔夜事件具备 Tier 3(事件驱动)级别的持续影响,应在输出中标记"建议补充进宏观记忆",并尽量给出建议归入的类别代码(参照该skill的 A–M 框架),由用户另行触发 `ashare-macro-context` 完成实际写入 (Tier 3 处理协议详见 `../../../references/tier3-event-handling.md`)。

## 所需本地输入

| 输入 | 内容 | 来源 |
|---|---|---|
| 持仓/自选股记录 | 用于判断隔夜消息是否与已有标的相关 | `ashare-intraday-review` 维护的 `positions`/`watchlist` 本地文件 |

| 宏观环境记忆 | 用于判断隔夜事件是否已被既有背景覆盖,或属于全新信息 | `ashare-macro-context` 维护的本地记忆文档 |
| 昨晚晚间复盘的隔夜需观察事项 | 用于核对这些事项的隔夜进展,避免遗漏 | `ashare-evening-review` 前一晚输出的"隔夜需观察事项"清单 |
| 隔夜美股/欧洲/全球行情 | 用于第1、2块内容 | 不规定具体获取方式 |
| 隔夜消息/今日已知日程 | 用于第3、4块内容 | 不规定具体获取方式 |

若持仓/自选股或宏观记忆文件不存在,提示用户先用对应skill建立,不要凭空假设内容。若昨晚晚间复盘留下的隔夜需观察事项本次仍无法给出结论,应继续保留进"今日观察清单",而不是默默丢弃。

## 四块推送内容

### 1. 隔夜美股
- 道指、纳指、标普三大指数收盘表现
- 中国概念股/ADR整体表现(对A股开盘情绪通常有参考价值,仅作参考,不做预测)
- 美债收益率、美元指数、原油等大宗商品隔夜变化
- 是否有重要美股公司财报/事件,可能关联国内某行业板块(只标注关联可能性,不展开分析)

### 2. 欧洲及全球其他主要市场
- 欧洲市场隔夜收盘表现(此时已完整收盘,数据完整)
- 隔夜是否有全球性宏观事件(央行表态、地缘政治事件等)
- **不含亚太**:亚太市场(日韩港)由昨晚 `ashare-evening-review` 覆盖,本块不重复;如需参照,只一句话指引"完整亚太数据见昨晚晚间复盘"

### 3. 隔夜消息汇总
- **时间窗口**:覆盖"昨晚晚间复盘之后 → 今早"的隔夜时段,承接前一晚晚间复盘的白天窗口,合起来才是完整24小时,不留缝
- **范围限定**:只收录与市场整体、宏观环境或持仓/自选股所属行业相关的信息,客观陈述,不做解读延伸,不是隔夜所有新闻的流水账
- **分类整理**(便于快速扫读,也便于与 `ashare-macro-context` 维度框架对应):按"国内政策与监管(隔夜如有)"/"国际重大事件"/"公司层面"/"行业与产业动态"分组摘要,而不是按时间顺序罗列 (完整分类标准见 `../../../references/message-classification.md`)
- **与持仓/自选股的关联标注**:整理时标注哪些消息与已有持仓/自选股直接相关,作为"今日观察清单"的来源依据之一
- **核对昨晚隔夜需观察事项**:逐条核对前一晚 `ashare-evening-review` 留下的"隔夜需观察事项"是否有隔夜进展或结论,有结论的在此说明,没有的并入今日观察清单继续观察
- 判断是否有 Tier 3 级别的新宏观事件需要标记"建议补充进 `ashare-macro-context` 记忆",并给出建议归入的类别代码(如 I_国际形势、K_全球主要央行政策等)

### 4. 今日已知日程 + 今日观察清单
- **今日已知日程**:经济数据发布、公司公告窗口、政策发布会、重要会议等——这属于"已知的不确定性",只标注时间和议题,不预测结果
- **今日观察清单**(衔接14:30盘中复盘):把今日需要关注的标的/主题列出,标注关注原因(来自隔夜消息、昨晚遗留或今日日程),仅作观察标记、非操作建议,交棒给当日 `ashare-intraday-review` 在盘中回顾并给出结论
- 中长期持仓如遇隔夜重大消息,明确写"已标记观察,按既有纪律线处理,不建议开盘前临时操作",而不是给出操作倾向

## 输出格式

按以下结构输出(模板见 `assets/morning_brief_template.md`),保持简洁,每条一两句话:

```
## 早间消息推送 [日期 8:00-8:20]

### 隔夜美股
- 三大指数表现 / 中国概念股表现 / 美债美元商品
- 重要美股财报/事件关联国内板块(如有,仅标注关联可能)

### 欧洲及全球其他市场
- 欧洲市场隔夜收盘表现(完整数据)
- 隔夜全球宏观事件(如有)
- (不含亚太——见昨晚晚间复盘)

### 隔夜消息汇总(昨晚晚间复盘→今早)
- 国内政策与监管:[隔夜如有]
- 国际重大事件:[摘要]
- 公司层面:[摘要,标注与持仓/自选股的关联]
- 行业与产业动态:[摘要]

### 短期机会发掘
- [标的]: 隔夜观察到的短期催化剂/事件驱动 → 为什么值得注意
- (若无)隔夜未发掘到值得注意的短期机会

### 昨晚隔夜观察事项核对
- [事项]: 隔夜进展/结论(已解释/并入今日观察清单)

### 今日已知日程
- [时间] [议题],仅标注待观察,不预测结果

### 宏观记忆更新提示
- (如有Tier3新事件)建议触发ashare-macro-context补充——建议归类:[类别代码]

### 今日观察清单(衔接14:30盘中复盘)
- [标的/主题]: 关注原因,仅作观察标记,非操作建议
```

## 与其他skill的关系

- **`ashare-evening-review`**:本skill承接其前一晚产出的"隔夜需观察事项"并核对隔夜进展;隔夜消息窗口与晚间复盘的白天窗口首尾相接,合成完整24小时闭环。短期机会方面,晚间复盘和早间推送各自从不同的时间窗口独立发掘新机会。
- **`ashare-intraday-review`**:本skill产出的"今日观察清单"交棒给当日 14:30-14:45 盘中复盘,在"宏观背景校准"和"待办"环节回顾并给出结论(已解释/仍观察/转入宏观记忆),不应让同一事项在多个skill之间无限循环却始终没有结论。三个skill共享同一套最低持有纪律,任何一方都不能单独据此突破。短期机会方面,早间推送和盘中复盘各自从不同的时间窗口(隔夜 vs 盘中)独立发掘新机会。
- **`ashare-macro-context`**:本skill不直接写入宏观长期记忆,只在发现 Tier 3 级新事件时提示用户另行触发该skill更新,并尽量给出建议归类。
- **`ashare-technical-analysis`**:若早间发现某持仓/自选股因隔夜消息需要更深入的技术面研判,提示用户另行触发该skill,不在20分钟窗口内展开。
