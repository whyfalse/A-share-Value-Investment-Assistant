# A股盘中复盘(14:30-14:45窗口)

## 适用边界(Scope)

本skill遵守 `../../../references/boundaries.md` 的全部共享边界，正文不再逐条复述。以下仅列本skill特有边界：

1. **这是"中继检查",不是"深度复盘"**:只有15分钟,目标是发现今天是否有需要在收盘前/今晚处理的异常,不是重新推导持仓逻辑。完整的多维技术面计算交给晚间复盘(它是唯一全量跑技术引擎的时段)。
2. **盘中不跑技术引擎**:盘中当日K线未收,现算指标会失真。本skill的技术深度来自"**技术位互动温度计**"——拿盘中现价,去和昨晚晚间复盘已算好的支撑/压力/纪律线做对照(承接关系与写法见 `../../../references/technical-depth-guide.md` §4.3),只描述"现价相对那些位的位置和距离",不现算新指标。

## 所需本地输入

复盘前需要读取以下本地文件(具体获取/同步方式不限定,由使用者维护):

| 文件 | 内容 | 模板 |
|---|---|---|
| 持仓记录 | 代码、名称、成本、仓位占比、止盈/止损/加仓线、买入日期、核心逻辑摘要 | `../../../templates/positions_template.json` |
| 自选股记录 | 代码、名称、预设买点条件、关注理由 | `../../../templates/watchlist_template.json` |
| 宏观环境记忆 | 由 `ashare-macro-context` skill维护的本地宏观背景文档 | 见该skill |
| 昨晚技术参考位 | 昨晚晚间复盘移交的各持仓支撑/压力/纪律线,作为今日盘中温度计的对照基准 | `../../../output/ashare-evening-review/evening_review_[昨日].md` |

> 此外,第4步需要实时盘中行情数据(指数、资金流向、板块及个股涨跌榜),按 `../../../references/data-source-priority.md` 选择工具:优先用环境中的专业金融数据工具实时获取,联网搜索仅作兜底,实时快照不存储。
> 昨晚晚间复盘报告若不存在(如周一早晨、节后首日、昨晚未复盘),则跳过技术位温度计的"对照昨晚位"部分,如实写"无昨晚参考位",仅做盘中量价定性,不回填、不编造历史测算位。

若某个文件不存在,提示用户先建立(可直接用对应template新建),不要凭空假设持仓内容。

## 六步复盘流程

> **启动前(取数准备)**:按 `../../../references/data-source-priority.md` 的"取数前配置准备"流程,先读 auto-memory 的 `data-source-config.json` 并做轻量校验;配置缺失或校验失败(关键专业工具消失/依赖缺失/`next_review_due` 过期)时,先接力调用 `ashare-data-source-config` 探测本地工具并重新编排配置,再开始取实时盘中数据。

### 1. 持仓检查(优先级最高)
- 对照持仓记录,逐一核对今日是否有显著异动(放量、触及涨跌停、尾盘大幅波动)
- 核对是否触及**用户自己预设**的止盈/止损/加仓线——这一步是"核对纪律是否被触发",不是临场重新判断逻辑是否成立
- **技术位互动温度计(承接昨晚晚间复盘的参考位,盘中不现算指标)**:对每只持仓,拿盘中现价去和昨晚算好的支撑/压力/纪律线做对照,每只输出一行:
  - 现价相对昨晚**支撑位**:上方/下方,距离百分比
  - 现价相对昨晚**压力位**:是否逼近/突破
  - 现价相对**用户纪律线**(止盈/止损):距离百分比,**触及才提示**(且只提示"已触及预设线,请核对是否执行你自己的纪律",不替用户下结论)
  - 一句**盘中量价定性**:放量/缩量配合现价方向(盘中量比可用,但K线形态不下定论)
  - 写法见 `../../../references/technical-depth-guide.md` §4.3;未满最低持有5日的持仓即便盘中跌破昨晚支撑,也只"标记观察/核对止损线",不建议卖出
- 是否有当日公司公告(分红、回购、业绩预告、股东变动、监管/立案等)需要消化
- 仓位结构是否因价格波动漂移,偏离用户自己设定的单票/行业占比上限

### 2. 自选股监控
- 是否有标的触及用户预设的买点条件(基于估值/质地的判断标准,不是因为今天上涨而临时起意)
- 是否出现明显异动,值得移入近期重点观察

### 3. 宏观背景校准
- 今天大盘/板块的异常,能否用 `ashare-macro-context` 记忆文档中已有的背景解释(尤其检查是否有对应的 Tier 3 事件驱动记录)
- 如果今天出现了明显的、记忆文档中还没有的 Tier 3 级新事件(央行决议、重要会议结论、关税/监管消息等),产出更新建议追加到 `../../../data/macro_updates_queue.json` (队列格式及协议详见 `../../../references/tier3-event-handling.md`),并初步判断它可能牵涉哪些持仓/自选股的长期逻辑链——仅作事实记录和标记观察,不据此下操作结论

### 4. 大盘/资金流向/板块情绪温度计
- **指数与情绪**:简要看一眼今日主要指数、涨跌停家数/涨跌家数比等,把握全市场冷暖
- **实时资金流向**:看当下(截至复盘时刻)主力/北向资金的净流入流出总量,以及资金主要流入和流出的板块(行业/概念),作为判断市场风格和热点切换的参照
- **主要个股涨跌**:扫一眼今日涨幅榜/跌幅榜前列、显著放量异动的个股(尤其与持仓/自选股同行业或同题材的),仅作市场情绪和热点的参照,不据此临时起意
- 本步的全部作用是参照系:帮助第1步判断"持仓个股的波动是自己的事还是大家的事"(系统性 beta 波动 vs 个股 alpha 信号),并为第5步短期机会发掘提供线索;资金流向和个股榜单都是实时快照,只描述现象、不预测方向、不构成操作建议
- 数据为实时盘中快照,按 `../../../references/data-source-priority.md` 选择工具(优先环境中的专业金融数据工具,联网搜索仅兜底)

### 5. 短期机会发掘(与1-2分开,单独成栏)
- **发掘**今日盘中新出现的、有明确短期催化剂的机会(题材消息、板块异动、事件驱动等),仅列出"今天观察到什么、为什么值得注意"
- 用催化剂框架描述,与中长期持仓的基本面/估值判断标准分栏、不混用

### 6. 待办与收尾
- 如发现需要操作的情况,写清楚"打算何时处理"(剩余交易时间内,还是收盘后/明天开盘),不在还剩几分钟时仓促建议下单
- 复盘中发现但无法当下下结论的疑点,列入"留待晚间/周末完整复盘"清单,不在这15分钟内强行展开

## 输出格式

按以下结构输出复盘摘要,保持简洁,每条一两句话,不展开成长篇分析:

```
## 盘中复盘 [日期 14:30-14:45]

### 持仓异动 / 纪律核对
- [标的]: 现象 → 是否触及预设线 → 建议关注/无需处理

### 技术位互动温度计(对照昨晚晚间复盘参考位,盘中不现算)
| 持仓 | 现价 | vs 昨晚支撑 | vs 昨晚压力 | 距止盈/止损 | 盘中量价 |
|---|---|---|---|---|---|
| [标的] | XX | 上方+X%/下方−X% | 逼近/已破/距X% | 距止损−X%(触及才提示) | 放量/缩量配合方向 |
- (无昨晚参考位时)昨晚未复盘/节后首日,无参考位 —— 仅做盘中量价定性
- (未满5日持仓)即便跌破昨晚支撑也只标记观察,不建议卖出

### 自选股提醒
- [标的]: 是否触及预设买点 / 是否异动

### 宏观背景校准
- 今日现象 ↔ 对应的已知宏观背景(或:发现新事件,已产出更新建议至队列——列出建议的条目)

### 大盘/资金流向/板块温度计
- 指数与涨跌停情绪一句话概括
- 实时资金流向:主力/北向净流入流出总量 + 主要流入板块、主要流出板块
- 主要个股涨跌:涨跌幅榜前列或显著放量异动(尤其与持仓/自选股相关的)一两句概括

### 短期机会发掘
- [标的]: 今日观察到的短期催化剂/异动 → 为什么值得注意

### 待办
- 今天剩余时间/收盘后需要处理的事项
- 留待晚间/周末完整复盘的疑点
```

## 输出保存

完成上述复盘后,除了在对话中展示,还必须把完整报告落盘保存到 `../../../output/ashare-intraday-review/` 文件夹下,文件名为 `intraday_review_[日期].md`(日期用当日,格式 `YYYY-MM-DD`,如 `intraday_review_2026-06-25.md`)。若文件夹不存在则先创建。同名文件已存在时直接覆盖(同一天重复复盘以最新一次为准)。

> 注:这里保存的是盘中复盘**报告**本身,用于留档和被晚间/周复盘回溯衔接。本skill"不存储"的原则仍然成立——指的是不把短期机会发掘的结果沉淀为长期清单、不改写持仓/自选股等状态文件,而不是不保存复盘报告。

## 与其他skill的关系

- **`ashare-macro-context`**:本skill的第3步直接读取它维护的本地记忆文档作为背景参照;若发现新的宏观事件,按 `../../../references/tier3-event-handling.md` 协议产出更新建议追加至队列,由 macro-context 统一消费处理。
- **`ashare-evening-review`**:本skill第1步的"技术位互动温度计"承接昨晚晚间复盘移交的技术参考位(各持仓支撑/压力/纪律线),盘中只做"现价 vs 昨晚位"的轻量对照,不重复跑技术引擎。盘中留下的未结疑点交棒给当晚晚间复盘收口。
- **`ashare-technical-analysis`**:盘中不调用该引擎(盘中K线未收,现算会失真);完整的多维技术指标计算由晚间复盘的第0步全量完成。若复盘中发现某持仓/自选股需要更深入的技术面专题研判,提示用户盘后另起一次该skill,不在15分钟窗口内展开。
- 本skill只读取持仓/自选股做核对;短期机会仅做当日**发掘式输出**。所有复盘技能(盘中/晚间/早间/周度)的短期机会均为独立发掘,各自从不同的时间窗口发现新机会。持仓/自选股的"新增、调仓、止盈止损线修改"由用户实际操作时同步更新对应本地文件。
