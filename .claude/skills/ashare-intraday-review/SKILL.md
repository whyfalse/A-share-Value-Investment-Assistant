# A股盘中复盘(14:30-14:45窗口)

## 适用边界(Scope) —— 必须遵守

本skill遵守 `../../../references/boundaries.md` 的全部共享边界（最低持有纪律、不构成投资建议、短期/中长期分栏、宏观背景引用边界等），以下仅列出本skill特有的边界：

1. **这是"中继检查",不是"深度复盘"**:只有15分钟,目标是发现今天是否有需要在收盘前/今晚处理的异常,不是重新推导持仓逻辑。深度复盘(基本面变化、技术面分析等)交给其他场景或晚间/周末进行。
2. **宏观背景只用于解释,不用于驱动**:本skill应读取 `ashare-macro-context` skill维护的本地宏观记忆文档,但只用它来解释"今天看到的现象是否对应已知的宏观背景",不能反过来用它去发起新的操作建议。

## 所需本地输入

复盘前需要读取以下本地文件(具体获取/同步方式不限定,由使用者维护):

| 文件 | 内容 | 模板 |
|---|---|---|
| 持仓记录 | 代码、名称、成本、仓位占比、止盈/止损/加仓线、买入日期、核心逻辑摘要 | `../../../templates/positions_template.json` |
| 自选股记录 | 代码、名称、预设买点条件、关注理由 | `../../../templates/watchlist_template.json` |
| 宏观环境记忆 | 由 `ashare-macro-context` skill维护的本地宏观背景文档 | 见该skill |

> 此外,第4步需要实时盘中行情数据(指数、资金流向、板块及个股涨跌榜),由使用者或运行环境的行情/资金流向工具实时获取,不规定具体来源,也不存储。

若某个文件不存在,提示用户先建立(可直接用对应template新建),不要凭空假设持仓内容。

## 六步复盘流程

### 1. 持仓检查(优先级最高)
- 对照持仓记录,逐一核对今日是否有显著异动(放量、触及涨跌停、尾盘大幅波动)
- 核对是否触及**用户自己预设**的止盈/止损/加仓线——这一步是"核对纪律是否被触发",不是临场重新判断逻辑是否成立
- 是否有当日公司公告(分红、回购、业绩预告、股东变动、监管/立案等)需要消化
- 仓位结构是否因价格波动漂移,偏离用户自己设定的单票/行业占比上限

### 2. 自选股监控
- 是否有标的触及用户预设的买点条件(基于估值/质地的判断标准,不是因为今天上涨而临时起意)
- 是否出现明显异动,值得移入近期重点观察

### 3. 宏观背景校准
- 今天大盘/板块的异常,能否用 `ashare-macro-context` 记忆文档中已有的背景解释(尤其检查是否有对应的 Tier 3 事件驱动记录)
- 如果今天出现了明显的、记忆文档中还没有的 Tier 3 级新事件(央行决议、重要会议结论、关税/监管消息等),产出更新建议追加到 `../../../data/macro_updates_queue.json` (队列格式及协议详见 `../../../references/tier3-event-handling.md`),并初步判断它可能牵涉哪些持仓/自选股的长期逻辑链——仅作事实记录和标记观察,不据此下操作结论
- 严格执行边界:这里只做解释和事实记录,不做操作建议

### 4. 大盘/资金流向/板块情绪温度计
- **指数与情绪**:简要看一眼今日主要指数、涨跌停家数/涨跌家数比等,把握全市场冷暖
- **实时资金流向**:看当下(截至复盘时刻)主力/北向资金的净流入流出总量,以及资金主要流入和流出的板块(行业/概念),作为判断市场风格和热点切换的参照
- **主要个股涨跌**:扫一眼今日涨幅榜/跌幅榜前列、显著放量异动的个股(尤其与持仓/自选股同行业或同题材的),仅作市场情绪和热点的参照,不据此临时起意
- 本步的全部作用是参照系:帮助第1步判断"持仓个股的波动是自己的事还是大家的事"(系统性 beta 波动 vs 个股 alpha 信号),并为第5步短期机会发掘提供线索;资金流向和个股榜单都是实时快照,只描述现象、不预测方向、不构成操作建议
- 数据为实时盘中快照,获取方式不限定(由使用者或运行环境的行情/资金流向工具决定)

### 5. 短期机会发掘(与1-2分开,单独成栏)
- **发掘**今日盘中新出现的、有明确短期催化剂的机会(题材消息、板块异动、事件驱动等),仅列出"今天观察到什么、为什么值得注意"
- 用催化剂框架描述,与中长期持仓的基本面/估值判断标准分栏、不混用

### 6. 待办与收尾
- 如发现需要操作的情况,写清楚"打算何时处理"(剩余交易时间内,还是收盘后/明天开盘),不在还剩几分钟时仓促建议下单
- 复盘中发现但无法当下下结论的疑点,列入"留待晚间/周末完整复盘"清单,不在这15分钟内强行展开

## 输出格式

按以下结构输出复盘摘要(模板见 `assets/review_output_template.md`),保持简洁,每条一两句话,不展开成长篇分析:

```
## 盘中复盘 [日期 14:30-14:45]

### 持仓异动 / 纪律核对
- [标的]: 现象 → 是否触及预设线 → 建议关注/无需处理

### 自选股提醒
- [标的]: 是否触及预设买点 / 是否异动

### 宏观背景校准
- 今日现象 ↔ 对应的已知宏观背景(或:发现新事件,已产出更新建议至队列——列出建议的条目)

### 大盘/资金流向/板块温度计
- 指数与涨跌停情绪一句话概括
- 实时资金流向:主力/北向净流入流出总量 + 主要流入板块、主要流出板块
- 主要个股涨跌:涨跌幅榜前列或显著放量异动(尤其与持仓/自选股相关的)一两句概括

### 短期机会发掘
- [标的]: 今日观察到的短期催化剂/异动 → 为什么值得注意

### 待办
- 今天剩余时间/收盘后需要处理的事项
- 留待晚间/周末完整复盘的疑点
```

## 与其他skill的关系

- **`ashare-macro-context`**:本skill的第3步直接读取它维护的本地记忆文档作为背景参照;若发现新的宏观事件,按 `../../../references/tier3-event-handling.md` 协议产出更新建议追加至队列,由 macro-context 统一消费处理。
- **`ashare-technical-analysis`**:若复盘中发现某持仓/自选股需要更深入的技术面研判(而不只是"是否异动"的检查),应提示用户另起一次该skill的分析,不在15分钟窗口内展开完整技术面分析。
- 本skill只读取持仓/自选股做核对;短期机会仅做当日**发掘式输出**。所有复盘技能(盘中/晚间/早间/周度)的短期机会均为独立发掘,各自从不同的时间窗口发现新机会。持仓/自选股的"新增、调仓、止盈止损线修改"由用户实际操作时同步更新对应本地文件。
