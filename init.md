# 初始化项目

你需要帮助用户初始化项目。

# 前置条件

## 必要条件

### 手动配置的条件

- 本地具备联网搜索或财经数据获取能力（MCP 或 API，例如 wind、akshare、东方财富choice），建议使用 MCP 服务

### 可以自行处理的条件

- python 3 环境
- Python 依赖：`pyyaml`（定时调度必需）、`akshare`（可选，用于交易日历判断与技术面分析）

## 可选条件

- 准备好个人的持仓股、自选股、关注板块等信息
- （如需定时邮件推送）用于接收报告的个人邮箱
- （如需定时邮件推送）用于发送邮件的邮箱，已开通 **SMTP 服务** 并记下 **授权密码**（授权密码将通过环境变量传入，不写入配置文件）

# 初始化步骤

## 准备必要条件

- 在`/scheduler/`下复制配置模板创建`config.yaml`（`cp scheduler/config.example.yaml scheduler/config.yaml`）。
- 读取 `/data/README.md`，按其文件清单从 `/templates/` 复制模板到 `/data/`，创建以下文件：
  - `positions.json`、`watchlist.json`、`sector_watchlist.json`（个人数据，可在后续步骤填写）
  - `macro_context.json`（所有技能都会读取的核心文件，从 `templates/macro_context_template.json` 复制；内容后续由 `ashare-macro-context` 技能填充维护）
- 询问用户是否准备了[手动配置的条件](#手动配置的条件)。
- 检查[可以自行处理的条件](#可以自行处理的条件)，如果缺少则自动安装。

## 是否启用邮件推送

询问用户是否需要启用发送邮件功能，如果需要则提示用户准备[可选条件](#可选条件)。

### 是

- 如果启用发送邮件功能：
  - 提示用户输入发送邮件的邮箱、收件邮箱，写入到`/scheduler/config.yaml`的`email.username`/`email.sender`/`email.recipients`。
  - **授权密码不写入配置文件**：`config.yaml`的`email.password`保留`${EMAIL_PASSWORD}`占位符，引导用户把授权密码设置为环境变量 `EMAIL_PASSWORD`（脚本运行时通过 `expand_env` 自动读取），避免密码明文入库。

### 否

- 提示用户后续如果要启用，可以自行配置。

## 是否填写个人数据

询问用户是否准备好个人持仓、自选股、关注板块等信息。

### 是

- 提示用户输入个人持仓信息，可以是文字描述，也可以是文本文件，分析后填写到`/data/positions.json`，其中股票代码、成本为必填字段，其他字段为可选填。
- 提示用户输入个人自选股信息，可以是文字描述，也可以是文本文件，分析后填写到`/data/watchlist.json`，其中股票代码为必填字段，其他字段为可选填。
- 提示用户输入个人关注板块信息，可以是文字描述，也可以是文本文件，分析后填写到`/data/sector_watchlist.json`，其中板块名称为必填字段，其他字段为可选填。
- 完成后，提示用户后续可以自行更改。

### 否

通知用户后续可以自行填写相关文件。

## 是否创建定时任务

询问用户是否需要创建定时任务，并告知用户将会在下面的时间点运行任务：

- 每个交易日的8:00进行早间消息推送
- 每个交易日的14:30进行盘中复盘
- 每个交易日的18:00进行晚间复盘
- 每周日10:00进行周复盘。

### 是

- 分别在上述时间点创建执行`/scheduler/run_review.py`脚本的定时任务，并提示用户后续可以自行更改。

### 否

提示用户后续可以自行创建定时任务。
