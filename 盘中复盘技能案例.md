# 盘中复盘（Intraday Review）

交易时段内快速决策分析工具。基于数据驱动（AKShare实时行情）和消息面（联网搜索），输出结构化的盘中复盘报告。

## 前置条件

- AKShare 已安装：`pip install akshare numpy pandas`
- 持仓文件已配置：`config/portfolio.json`
- 当前处于A股交易时段（9:30-15:00）效果最佳，非交易时段可获取最近交易日的盘中快照

## 分析流程

按以下步骤依次执行，每一步完成后汇报进度再进行下一步。

### 第一步：读取用户持仓

1. 使用 Read 工具读取持仓文件。如果用户通过 `$ARGUMENTS` 指定了路径则使用该路径，否则默认读取 `config/portfolio.json`
2. 确认持仓列表，向用户展示并快速确认："本次盘中复盘将分析以下持仓：XX股（代码XXX）、YY股（代码YYY）…"

### 第二步：获取实时行情数据

运行数据获取脚本。**注意**：脚本必须从项目根目录运行：

```bash
cd d:/ai-design/ai-market-review && python .claude/skills/intraday-review/scripts/intraday_data.py --portfolio config/portfolio.json
```

> **重要**：脚本输出为完整JSON。出于篇幅考虑，在报告中只展示关键数据和解读。完整JSON保留在报告末尾的"原始数据"折叠区域。

脚本返回的数据包含以下维度（筛选关键字段用于分析，详细数据结构见脚本输出）：

**指数行情（index_data）**：
- 上证指数 / 深证成指 / 创业板指 / 科创50 的最新收盘价、涨跌幅、成交量
- 与前一日对比的涨跌幅度

**市场情绪（market_breadth）**：
- 涨跌家数比、涨停/跌停家数
- 上涨家数 / 下跌家数 / 平盘家数

**北向资金（north_bound_flow）**：
- 北向资金盘中净买入/卖出金额
- 沪股通vs深股通流向差异

**板块排名（sector_leaders）**：
- 涨幅前5和跌幅前5的板块名称及涨跌幅
- 板块资金是否持续流入

**涨停板（limit_up_pool）**：
- 涨停股票数量
- 涨停个股列表（前30只）

**持仓个股实时行情（portfolio_analysis）**：
- 每只持仓股的涨跌幅、换手率、量比、振幅
- 最新价、盈亏比例、盈亏金额
- 与止盈止损价位的距离

**市场资金流（market_fund_flow）**：
- 主力资金净流入/流出

### 第三步：联网搜索盘中消息

使用 WebSearch 工具进行**多轮搜索**，每轮关注不同维度：

**第一轮：宏观政策消息**
```
搜索词：今日 A股 央行 证监会 政策 消息
```
摘取最近2-3条重要政策或监管消息。

**第二轮：持仓板块行业动态**
- 对持仓股票所属板块，分别搜索当日行业消息
- 例如持仓中有半导体股，则搜索：`半导体 行业 今日 新闻`

**第三轮：持仓个股盘中公告**
- 对每只持仓股，搜索其今日是否有公告发布
- 搜索词示例：`<股票名称> 公告 重大事项`

**第四轮：隔夜国际市场影响**
```
搜索词：美股 收盘 今日 A50 期货 亚太股市
```

### 第四步：综合分析与操作建议

基于以上数据，生成结构化盘中复盘报告。报告结构和每部分的写作要求如下：

## 异常处理

- **AKShare接口失败**：在报告中标注"数据获取失败"，尝试使用WebSearch获取替代数据
- **持仓文件不存在**：提示用户先创建持仓配置，引导运行 `cp config/portfolio.json.example config/portfolio.json` 或手动创建
- **非交易时段**：提醒用户当前非交易时段，数据为最近交易日盘中快照，建议使用晚间复盘
- **数据不完整**：部分字段缺失时标注"N/A"，不影响其他分析
